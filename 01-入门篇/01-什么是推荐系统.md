# 第一章：什么是推荐系统

> 推荐系统入门第一课，建立整体认知

---

## 1.1 什么是推荐系统

**推荐系统（Recommender System）** 是一种信息过滤系统，通过分析用户的历史行为、兴趣偏好和上下文信息，向用户推荐可能感兴趣的物品（商品、内容、服务等）。

### 简单来说

> 推荐系统就是**猜你喜欢什么**，然后**推荐给你**的系统。

### 经典案例

| 平台 | 推荐内容 | 典型场景 |
|------|---------|---------|
| 淘宝/京东 | 商品 | "猜你喜欢" |
| 抖音/B 站 | 视频 | "为你推荐" |
| 网易云音乐 | 音乐 | "每日推荐" |
| 今日头条 | 新闻 | "推荐"频道 |
| Netflix | 电影 | "因为你看了 XXX" |

---

## 1.2 为什么需要推荐系统

### 1.2.1 信息过载问题

随着互联网发展，信息呈现爆炸式增长：

- **电商：** 淘宝有**10 亿+**商品
- **视频：** YouTube 每分钟上传**500 小时**视频
- **音乐：** Spotify 有**8000 万+**歌曲
- **新闻：** 每天产生**数百万篇**新闻

**问题：** 用户如何在海量信息中找到自己需要的内容？

**答案：** 推荐系统！

### 1.2.2 推荐系统的价值

#### 对用户

- ✅ **节省时间** — 快速找到感兴趣的内容
- ✅ **发现惊喜** — 发现不知道但喜欢的内容
- ✅ **个性化体验** — 千人千面，因人而异

#### 对平台

- ✅ **提升用户粘性** — 用户停留时间更长
- ✅ **增加收入** — 提高转化率、GMV
- ✅ **竞争优势** — 更好的用户体验

#### 对商家/创作者

- ✅ **精准触达** — 找到目标用户
- ✅ **降低获客成本** — 流量更精准
- ✅ **长尾效应** — 小众内容也能被发现

## 1.3 推荐系统 vs 搜索引擎

很多人会混淆推荐系统和搜索引擎，它们有什么区别？

| 维度 | 推荐系统 | 搜索引擎 |
|------|---------|---------|
| **目标** | 发现用户可能感兴趣的内容 | 找到用户明确需要的内容 |
| **输入** | 用户画像、历史行为 | 搜索关键词 |
| **输出** | 个性化推荐列表 | 搜索结果列表 |
| **主动性** | 主动推荐 | 被动响应 |
| **典型场景** | "猜你喜欢" | "搜索 XXX" |

### 举例说明

- **推荐系统：** 打开抖音，自动播放你可能喜欢的视频
- **搜索引擎：** 在 Google 搜索"推荐系统教程"

### 融合趋势

现代平台往往**同时使用**两种方式：

- 淘宝：既有"猜你喜欢"（推荐），也有搜索框（搜索）
- B 站：既有"推荐"频道，也有搜索功能

## 1.4 推荐系统基本架构

一个典型的推荐系统包含以下核心模块：

```
┌─────────────────────────────────────────────────────────┐
│                      用户界面                            │
│                   (App/Web/小程序)                        │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│                      数据采集                            │
│            (行为日志、用户信息、物品信息)                   │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│                      数据处理                            │
│              (清洗、特征工程、样本构建)                     │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│                      召回层                              │
│          (从百万/千万物品中筛选千级别候选集)                 │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│                      排序层                              │
│           (对候选集精细排序，输出百级别推荐)                  │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│                      重排层                              │
│         (去重、多样性控制、业务规则调整)                     │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│                      推荐结果                            │
│                (展示给用户)                               │
└─────────────────────────────────────────────────────────┘
```

### 各层详解

#### 1. 数据采集

收集用户和物品的各种信息：

- **用户信息：** 年龄、性别、地域、兴趣标签
- **物品信息：** 类别、标签、价格、描述
- **行为数据：** 点击、浏览、购买、评分、收藏

#### 2. 召回层

从海量物品中快速筛选出**千级别**候选集：

- **协同过滤召回** — 相似用户/物品
- **内容召回** — 基于标签/类别
- **热门召回** —  trending 物品
- **地理位置召回** — 附近的内容

#### 3. 排序层

对候选集进行**精细排序**，输出百级别推荐：

- **机器学习模型** — LR、GBDT、XGBoost
- **深度学习模型** — Wide&Deep、DeepFM、DIN
- **多目标优化** — 点击率 + 转化率 + 时长

#### 4. 重排层

最后的结果调整：

- **去重** — 避免重复物品
- **多样性** — 避免单一类型
- **业务规则** — 广告插入、新品扶持
- **用户体验** — 避免连续相似内容

## 1.5 行业应用案例

### 1.5.1 电商推荐（淘宝/京东）

**场景：** 商品推荐

**特点：**
- 物品数量：10 亿+
- 用户数量：10 亿+
- 实时性要求：高
- 目标：GMV 最大化

**典型应用：**
- 首页"猜你喜欢"
- 购物车推荐
- 订单完成页推荐
- 关联推荐（买了又买）

### 1.5.2 视频推荐（抖音/B 站）

**场景：** 视频内容推荐

**特点：**
- 物品数量：数亿视频
- 用户停留时间长
- 实时反馈（完播率、点赞）
- 目标：用户时长最大化

**典型应用：**
- 抖音"推荐"页
- B 站"推荐"频道
- Netflix"为你推荐"

### 1.5.3 音乐推荐（网易云/Spotify）

**场景：** 音乐推荐

**特点：**
- 物品数量：数千万歌曲
- 用户兴趣相对稳定
- 场景化推荐（运动、睡眠）
- 目标：播放时长、付费转化

**典型应用：**
- 网易云"每日推荐"
- Spotify"Discover Weekly"
- 跑步/睡眠歌单

### 1.5.4 新闻推荐（今日头条）

**场景：** 新闻资讯推荐

**特点：**
- 物品时效性强
- 用户兴趣多变
- 实时性要求极高
- 目标：点击率、阅读时长

**典型应用：**
- 今日头条"推荐"频道
- 腾讯新闻"猜你喜欢"
- 百度 APP 信息流
