# 第三章：推荐系统评估指标

> 如何科学评价一个推荐系统的好坏

---

## 3.1 为什么需要评估指标

在推荐系统中，评估指标至关重要：

### 3.1.1 评估的作用

1. **衡量效果** — 量化推荐系统的好坏
2. **指导优化** — 明确优化方向
3. **A/B 测试** — 对比不同算法/策略
4. **业务对齐** — 将技术指标与业务目标关联

### 3.1.2 评估的层次

```
用户体验层 ←→ 算法性能层 ←→ 业务价值层
```

| 层次 | 指标示例 | 关注点 |
|------|---------|--------|
| **用户体验** | 满意度、NPS | 用户主观感受 |
| **算法性能** | Precision、Recall、NDCG | 推荐准确性 |
| **业务价值** | GMV、留存、付费率 | 商业目标达成 |

## 3.2 准确率指标

### 3.2.1 Precision（查准率）

**定义：** 推荐的内容中有多少是用户真正感兴趣的

**公式：**
```
Precision = TP / (TP + FP)
```

**说明：**
- TP（True Positive）：推荐且用户感兴趣
- FP（False Positive）：推荐但用户不感兴趣

**示例：**
```
推荐了 10 个商品
用户点击了其中 3 个
Precision = 3/10 = 30%
```

**代码实现：**
```python
def precision(recommended, relevant):
    """
    计算 Precision
    
    Args:
        recommended: 推荐列表
        relevant: 用户实际感兴趣的物品
    
    Returns:
        Precision 值
    """
    hits = len(set(recommended) & set(relevant))
    return hits / len(recommended) if recommended else 0

# 示例
recommended = ['A', 'B', 'C', 'D', 'E']
relevant = ['A', 'C', 'F', 'G']
print(f"Precision: {precision(recommended, relevant):.2%}")  # 40%
```

**适用场景：**
- 推荐列表长度固定
- 关注推荐质量而非数量

### 3.2.2 Recall（查全率）

**定义：** 用户感兴趣的内容中有多少被推荐出来了

**公式：**
```
Recall = TP / (TP + FN)
```

**说明：**
- FN（False Negative）：用户感兴趣但未推荐

**示例：**
```
用户感兴趣的商品有 20 个
推荐了其中 5 个
Recall = 5/20 = 25%
```

**代码实现：**
```python
def recall(recommended, relevant):
    """
    计算 Recall
    """
    hits = len(set(recommended) & set(relevant))
    return hits / len(relevant) if relevant else 0

# 示例
recommended = ['A', 'B', 'C', 'D', 'E']
relevant = ['A', 'C', 'F', 'G']
print(f"Recall: {recall(recommended, relevant):.2%}")  # 50%
```

**适用场景：**
- 关注覆盖率
- 不希望遗漏用户感兴趣的内容

### 3.2.3 F1 Score（F1 分数）

**定义：** Precision 和 Recall 的调和平均

**公式：**
```
F1 = 2 * Precision * Recall / (Precision + Recall)
```

**代码实现：**
```python
def f1_score(precision, recall):
    """
    计算 F1 Score
    """
    if precision + recall == 0:
        return 0
    return 2 * precision * recall / (precision + recall)

# 示例
p = 0.4  # Precision
r = 0.5  # Recall
print(f"F1 Score: {f1_score(p, r):.2%}")  # 44.4%
```

**适用场景：**
- 需要平衡 Precision 和 Recall
- 单一指标评估

### 3.2.4 Precision@K 和 Recall@K

**定义：** 只考虑前 K 个推荐结果

**公式：**
```
Precision@K = TP@K / K
Recall@K = TP@K / |Relevant|
```

**代码实现：**
```python
def precision_at_k(recommended, relevant, k):
    """
    计算 Precision@K
    """
    top_k = recommended[:k]
    hits = len(set(top_k) & set(relevant))
    return hits / k if k > 0 else 0

def recall_at_k(recommended, relevant, k):
    """
    计算 Recall@K
    """
    top_k = recommended[:k]
    hits = len(set(top_k) & set(relevant))
    return hits / len(relevant) if relevant else 0

# 示例
recommended = ['A', 'B', 'C', 'D', 'E', 'F', 'G']
relevant = ['A', 'C', 'F', 'H', 'I']
k = 5

print(f"Precision@{k}: {precision_at_k(recommended, relevant, k):.2%}")  # 40%
print(f"Recall@{k}: {recall_at_k(recommended, relevant, k):.2%}")  # 60%
```

**适用场景：**
- 推荐列表有长度限制
- 只关注头部推荐质量

## 3.3 排序质量指标

### 3.3.1 NDCG（归一化折损累积增益）

**定义：** 考虑推荐顺序的指标，越靠前的推荐权重越高

**核心概念：**
- **CG（Cumulative Gain）** — 累积增益
- **DCG（Discounted Cumulative Gain）** — 折损累积增益
- **NDCG（Normalized DCG）** — 归一化 DCG

**公式：**
```
DCG@K = Σ(i=1 to K) rel_i / log2(i+1)
NDCG@K = DCG@K / IDCG@K
```

**代码实现：**
```python
import math

def dcg_at_k(relevances, k):
    """
    计算 DCG@K
    
    Args:
        relevances: 相关性分数列表（按推荐顺序）
        k: 考虑前 K 个
    """
    relevances = relevances[:k]
    dcg = 0
    for i, rel in enumerate(relevances):
        dcg += rel / math.log2(i + 2)  # i+2 因为 i 从 0 开始
    return dcg

def idcg_at_k(relevances, k):
    """
    计算 IDCG@K（理想 DCG）
    """
    # 按相关性降序排列
    sorted_relevances = sorted(relevances, reverse=True)
    return dcg_at_k(sorted_relevances, k)

def ndcg_at_k(relevances, k):
    """
    计算 NDCG@K
    """
    dcg = dcg_at_k(relevances, k)
    idcg = idcg_at_k(relevances, k)
    return dcg / idcg if idcg > 0 else 0

# 示例
# 推荐了 5 个物品，用户相关性评分（0-5 分）
# 推荐顺序：[5, 3, 1, 4, 2]
relevances = [5, 3, 1, 4, 2]
k = 5

print(f"NDCG@{k}: {ndcg_at_k(relevances, k):.4f}")  # 0.8572
```

**适用场景：**
- 推荐顺序很重要
- 相关性有程度区分（不只是相关/不相关）

### 3.3.2 MAP（Mean Average Precision）

**定义：** 多个查询的平均 AP

**公式：**
```
AP = Σ(Precision@k * rel(k)) / |Relevant|
MAP = (1/|Q|) * Σ AP(q)
```

**代码实现：**
```python
def average_precision(recommended, relevant):
    """
    计算 Average Precision
    """
    precisions = []
    hits = 0
    for i, item in enumerate(recommended):
        if item in relevant:
            hits += 1
            precisions.append(hits / (i + 1))
    
    return sum(precisions) / len(relevant) if relevant else 0

def mean_average_precision(query_results):
    """
    计算 MAP
    
    Args:
        query_results: [(recommended, relevant), ...]
    """
    aps = [average_precision(rec, rel) for rec, rel in query_results]
    return sum(aps) / len(aps) if aps else 0

# 示例
query_results = [
    (['A', 'B', 'C'], ['A', 'C']),  # 查询 1
    (['D', 'E', 'F'], ['D', 'F']),  # 查询 2
    (['G', 'H', 'I'], ['G']),       # 查询 3
]

print(f"MAP: {mean_average_precision(query_results):.4f}")
```

**适用场景：**
- 多用户/多查询评估
- 关注整体平均表现

## 3.4 预测 accuracy 指标

### 3.4.1 MAE（平均绝对误差）

**定义：** 预测评分与真实评分的平均绝对差

**公式：**
```
MAE = (1/n) * Σ|ŷ_i - y_i|
```

**代码实现：**
```python
import numpy as np

def mae(y_true, y_pred):
    """
    计算 MAE
    """
    return np.mean(np.abs(np.array(y_true) - np.array(y_pred)))

# 示例
y_true = [5, 4, 3, 5, 2]  # 真实评分
y_pred = [4.5, 4.2, 2.8, 4.8, 2.5]  # 预测评分

print(f"MAE: {mae(y_true, y_pred):.4f}")  # 0.34
```

**特点：**
- 对异常值不敏感
- 易于理解和解释

### 3.4.2 RMSE（均方根误差）

**定义：** 预测评分与真实评分的均方根差

**公式：**
```
RMSE = sqrt((1/n) * Σ(ŷ_i - y_i)²)
```

**代码实现：**
```python
def rmse(y_true, y_pred):
    """
    计算 RMSE
    """
    return np.sqrt(np.mean((np.array(y_true) - np.array(y_pred)) ** 2))

# 示例
y_true = [5, 4, 3, 5, 2]
y_pred = [4.5, 4.2, 2.8, 4.8, 2.5]

print(f"RMSE: {rmse(y_true, y_pred):.4f}")  # 0.41
```

**特点：**
- 对大误差更敏感
- 常用于评分预测任务

### 3.4.3 MAE vs RMSE

| 指标 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **MAE** | 稳健、易解释 | 对大误差不敏感 | 一般预测 |
| **RMSE** | 强调大误差 | 对异常值敏感 | 要求高精度 |

## 3.5 覆盖率指标

### 3.5.1 物品覆盖率

**定义：** 被推荐过的物品占总物品的比例

**公式：**
```
Coverage = |Recommended Items| / |All Items|
```

**代码实现：**
```python
def item_coverage(recommended_items, all_items):
    """
    计算物品覆盖率
    """
    return len(set(recommended_items)) / len(all_items)

# 示例
recommended_items = ['A', 'B', 'C', 'D', 'E', 'A', 'B']
all_items = ['A', 'B', 'C', 'D', 'E', 'F', 'G', 'H', 'I', 'J']

print(f"Coverage: {item_coverage(recommended_items, all_items):.2%}")  # 50%
```

### 3.5.2 用户覆盖率

**定义：** 收到推荐的用户占总用户的比例

**意义：**
- 衡量推荐系统的普及程度
- 发现冷启动用户

## 3.6 多样性指标

### 3.6.1 类别多样性

**定义：** 推荐物品的类别分布

**计算方式：**
```
Diversity = 1 - Similarity
Similarity = 平均物品间相似度
```

**代码实现：**
```python
def diversity(recommended_categories):
    """
    计算多样性（基于类别分布熵）
    """
    from collections import Counter
    import math
    
    # 统计类别分布
    counter = Counter(recommended_categories)
    total = len(recommended_categories)
    
    # 计算熵
    entropy = 0
    for count in counter.values():
        p = count / total
        entropy -= p * math.log2(p)
    
    # 归一化（最大熵为 log2(类别数)）
    max_entropy = math.log2(len(counter)) if counter else 1
    return entropy / max_entropy if max_entropy > 0 else 0

# 示例
categories = ['电影', '电影', '音乐', '书籍', '电影', '音乐', '游戏', '书籍']
print(f"Diversity: {diversity(categories):.4f}")  # 0.94
```

### 3.6.2 新颖性

**定义：** 推荐用户不知道的物品

**计算方式：**
```
Novelty = -log2(p(item))
```

**意义：**
- 避免只推荐热门物品
- 帮助用户发现新内容

### 3.6.3 惊喜度

**定义：** 推荐用户意想不到的物品

**特点：**
- 比新颖性更强
- 需要平衡相关性

## 3.7 业务指标

### 3.7.1 点击率（CTR）

**定义：** 点击次数 / 曝光次数

**公式：**
```
CTR = Clicks / Impressions
```

**意义：**
- 最基础的指标
- 反映用户兴趣匹配度

**典型值：**
- 电商：3-8%
- 新闻：5-15%
- 视频：10-30%

### 3.7.2 转化率（CVR）

**定义：** 转化次数 / 点击次数

**公式：**
```
CVR = Conversions / Clicks
```

**意义：**
- 反映推荐质量
- 直接关联收入

### 3.7.3 GMV（成交总额）

**定义：** 推荐带来的成交金额

**计算：**
```
GMV = Σ(成交商品金额)
```

**意义：**
- 电商核心指标
- 直接反映商业价值

### 3.7.4 用户留存

**定义：** 次日/7 日/30 日后继续使用的用户比例

**意义：**
- 反映长期价值
- 避免短期优化

## 3.8 指标选择指南

### 3.8.1 按场景选择

| 场景 | 核心指标 | 辅助指标 |
|------|---------|---------|
| **电商** | GMV、CVR | CTR、客单价 |
| **视频** | 观看时长、完播率 | CTR、互动率 |
| **音乐** | 播放时长、收藏率 | 分享率、付费率 |
| **新闻** | CTR、阅读时长 | 分享率、评论率 |
| **社交** | 互动率、留存率 | 好友增长率 |

### 3.8.2 按阶段选择

| 阶段 | 核心指标 | 原因 |
|------|---------|------|
| **冷启动** | 覆盖率、留存率 | 扩大用户基础 |
| **成长期** | CTR、时长 | 提升用户体验 |
| **成熟期** | GMV、付费率 | 商业价值最大化 |

### 3.8.3 多目标平衡

推荐系统往往需要平衡多个目标：

```
短期收益（CTR）←→ 长期价值（留存）
用户体验 ←→ 商业收入
准确性 ←→ 多样性
```

**解决方案：**
- 多任务学习
- 加权目标函数
- Pareto 优化

## 3.9 评估方法

### 3.9.1 离线评估

**方法：** 使用历史数据评估

**优点：**
- 快速迭代
- 成本低
- 可重复

**缺点：**
- 与线上有差距
- 无法评估新颖性

**常用指标：** Precision、Recall、NDCG、RMSE

### 3.9.2 在线评估（A/B 测试）

**方法：** 线上分流对比

**优点：**
- 真实用户反馈
- 直接反映业务价值

**缺点：**
- 成本高
- 周期长
- 有风险

**流程：**
```
假设 → 实验设计 → 分流 → 数据收集 → 统计分析 → 决策
```

### 3.9.3 用户调研

**方法：** 问卷、访谈、可用性测试

**优点：**
- 主观感受
- 发现深层问题

**缺点：**
- 样本量小
- 主观偏差
