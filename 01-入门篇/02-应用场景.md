# 第二章：推荐系统应用场景

> 深入了解不同行业的推荐系统特点和解决方案

---

## 2.1 电商推荐

### 2.1.1 典型场景

电商是推荐系统应用最成熟的领域之一，几乎贯穿用户购物的整个流程。

#### 1. 首页"猜你喜欢"

**位置：** APP 首页核心位置

**特点：**
- 流量最大，曝光最高
- 个性化程度要求高
- 实时更新用户兴趣

**常见做法：**
- 多路召回（协同过滤 + 内容 + 热门）
- 深度学习排序（DeepFM、DIN）
- 实时特征更新

#### 2. 商品详情页推荐

**位置：** 商品详情页底部/侧边

**典型文案：**
- "看了又看"
- "相似商品"
- "搭配购"

**常见做法：**
- 基于物品的协同过滤（ItemCF）
- 商品相似度计算（图片、文本、属性）
- 价格区间控制

#### 3. 购物车推荐

**位置：** 购物车页面

**典型文案：**
- "再买 X 元免运费"
- "搭配购买更优惠"

**常见做法：**
- 关联规则挖掘（Apriori、FP-Growth）
- 价格敏感度分析
- 运费门槛计算

#### 4. 订单完成页推荐

**位置：** 支付成功页面

**典型文案：**
- "买了该商品的人还买了"
- "搭配配件"

**常见做法：**
- 购物篮分析
- 高频搭配挖掘
- 冲动消费引导

### 2.1.2 电商推荐特点

| 特点 | 说明 | 应对方案 |
|------|------|---------|
| **物品量大** | 10 亿 + 商品 | 多路召回 + 分层过滤 |
| **用户量大** | 10 亿 + 用户 | 分布式计算 + 用户分群 |
| **实时性高** | 分钟级更新 | 流式计算 + 在线学习 |
| **目标多元** | GMV、点击、转化 | 多目标优化 |
| **冷启动严重** | 新品、新用户多 | 探索利用 + 迁移学习 |

### 2.1.3 核心指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| **CTR** | 点击率 = 点击/曝光 | 3-8% |
| **CVR** | 转化率 = 购买/点击 | 5-15% |
| **GMV** | 成交总额 | 持续增长 |
| **UV 价值** | 每用户贡献 GMV | 持续提升 |
| **复购率** | 再次购买用户占比 | 30%+ |

## 2.2 视频推荐

### 2.2.1 典型场景

#### 1. 信息流推荐（抖音/TikTok）

**特点：**
- 沉浸式体验，单列全屏
- 强依赖前 3 秒完播率
- 实时反馈（点赞、评论、转发）

**常见做法：**
- 视频内容理解（画面、音频、文本）
- 用户兴趣实时捕捉
- 完播率预估模型

**核心指标：**
- 完播率
- 观看时长
- 互动率（点赞、评论、转发）

#### 2. 推荐频道（B 站/YouTube）

**特点：**
- 双列展示，用户主动选择
- 长视频为主，决策成本高
- UP 主/频道维度推荐

**常见做法：**
- 创作者影响力建模
- 视频序列推荐
- 订阅关系利用

#### 3. 播放页推荐（Netflix）

**位置：** 播放结束后

**典型文案：**
- "因为你看了 XXX"
- "继续观看"

**常见做法：**
- 观看序列建模
- 剧情连贯性分析
- 续集/同系列推荐

### 2.2.2 视频推荐特点

| 特点 | 说明 | 应对方案 |
|------|------|---------|
| **内容消费时间长** | 单视频数分钟到数小时 | 时长预估模型 |
| **内容时效性强** | 热点视频爆发快 | 实时 trending 检测 |
| **冷启动严重** | 每天大量新视频上传 | 内容理解 + 流量扶持 |
| **马太效应** | 头部视频占据大部分流量 | 多样性控制 + 长尾扶持 |
| **多目标优化** | 时长、互动、付费 | 多任务学习 |

### 2.2.3 核心指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| **播放时长** | 人均观看时长 | 持续增长 |
| **完播率** | 完整观看占比 | 30%+ |
| **互动率** | 点赞 + 评论 + 转发 | 5%+ |
| **次日留存** | 次日继续观看 | 40%+ |
| **付费转化** | 会员购买率 | 10%+ |

## 2.3 音乐推荐

### 2.3.1 典型场景

#### 1. 每日推荐（网易云/Spotify）

**特点：**
- 每日更新，固定 30 首左右
- 兼顾熟悉和探索
- 用户期待值高

**常见做法：**
- 用户长期兴趣建模
- 探索与利用平衡
- 歌曲多样性控制

#### 2. 场景推荐

**典型场景：**
- 跑步歌单
- 睡眠歌单
- 学习歌单
- 驾车歌单

**技术要点：**
- 场景识别（时间、地点、活动）
- BPM 匹配（跑步节奏）
- 情绪分析

#### 3. 相似歌曲推荐

**位置：** 播放页

**典型文案：**
- "相似歌曲"
- "喜欢这首歌的人也喜欢"

**技术要点：**
- 音频特征分析（旋律、节奏、音色）
- 歌词内容分析
- 用户协同过滤

### 2.3.2 音乐推荐特点

| 特点 | 说明 | 应对方案 |
|------|------|---------|
| **用户兴趣稳定** | 音乐品味相对固定 | 长期兴趣建模 |
| **重复消费** | 同一首歌反复听 | 播放频率控制 |
| **场景化强** | 不同场景听不同音乐 | 场景识别 |
| **长尾明显** | 小众音乐也有受众 | 多样性推荐 |
| **情感因素** | 音乐与情绪强相关 | 情绪分析 |

### 2.3.3 核心指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| **播放时长** | 人均收听时长 | 持续增长 |
| **收藏率** | 收藏/播放 | 10%+ |
| **分享率** | 分享/播放 | 3%+ |
| **付费转化** | 会员购买率 | 15%+ |
| **满意度** | 红心/播放 | 20%+ |

## 2.4 新闻推荐

### 2.4.1 典型场景

#### 1. 信息流推荐（今日头条）

**特点：**
- 内容时效性极强
- 用户兴趣多变
- 实时热点驱动

**技术要点：**
- 实时新闻理解
- 热点检测和追踪
- 用户兴趣实时更新

#### 2. 搜索推荐

**位置：** 搜索结果页

**特点：**
- 主动搜索 + 推荐结合
- 查询意图理解
- 权威性和时效性平衡

#### 3. 推送通知

**特点：**
- 打扰用户，需谨慎
- 高时效性内容
- 个性化推送

**技术要点：**
- 重要性评估
- 推送时机选择
- 用户打扰度控制

### 2.4.2 新闻推荐特点

| 特点 | 说明 | 应对方案 |
|------|------|---------|
| **时效性极强** | 新闻价值随时间衰减 | 时间衰减因子 |
| **内容生命周期短** | 大部分新闻 1-2 天过期 | 实时召回 |
| **热点集中** | 重大新闻所有人关注 | 热点 + 个性化融合 |
| **质量参差** | 存在标题党、假新闻 | 质量评估 + 人工审核 |
| **信息茧房** | 只看感兴趣的内容 | 多样性控制 |

### 2.4.3 核心指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| **CTR** | 点击率 | 5%+ |
| **阅读时长** | 平均阅读时间 | 1 分钟 + |
| **读完率** | 完整阅读占比 | 30%+ |
| **分享率** | 分享/阅读 | 3%+ |
| **评论率** | 评论/阅读 | 1%+ |

## 2.5 社交推荐

### 2.5.1 典型场景

#### 1. 好友推荐（微信/微博）

**推荐依据：**
- 共同好友
- 通讯录匹配
- 兴趣相似
- 地理位置接近

**技术要点：**
- 社交图谱分析
- 隐私保护
- 推荐频率控制

#### 2. 内容推荐（朋友圈/微博）

**特点：**
- 基于社交关系
- 时间线 + 推荐混合
- 强关系优先

**技术要点：**
- 关系强度建模
- 内容质量评估
- 时间衰减

#### 3. 群组推荐

**推荐依据：**
- 兴趣匹配
- 好友所在群
- 地理位置

### 2.5.2 社交推荐特点

| 特点 | 说明 | 应对方案 |
|------|------|---------|
| **隐私敏感** | 涉及用户社交关系 | 隐私保护 + 用户授权 |
| **关系复杂** | 强弱关系、双向单向 | 关系图谱建模 |
| **网络效应** | 推荐影响社交网络结构 | 长期价值优化 |
| **冷启动** | 新用户社交关系少 | 兴趣推荐 + 通讯录 |

## 2.6 本地生活推荐

### 2.6.1 典型场景

#### 1. 餐厅推荐（大众点评/美团）

**推荐依据：**
- 地理位置
- 口味偏好
- 消费水平
- 时间段

**技术要点：**
- LBS 推荐
- 营业时间匹配
- 排队情况预估

#### 2. 酒店推荐（携程/Booking）

**推荐依据：**
- 目的地
- 出行目的（商务/旅游）
- 价格区间
- 设施偏好

### 2.6.2 本地生活推荐特点

| 特点 | 说明 | 应对方案 |
|------|------|---------|
| **强地理位置** | 距离是关键因素 | LBS 推荐 |
| **低频消费** | 决策周期长 | 跨品类推荐 |
| **高客单价** | 决策谨慎 | 评价 + 攻略引导 |
| **时效性** | 营业时间和库存 | 实时数据同步 |

## 2.7 各平台对比

### 2.7.1 推荐策略对比

| 平台 | 核心目标 | 主要算法 | 特色功能 |
|------|---------|---------|---------|
| **淘宝** | GMV 最大化 | 多目标深度学习 | 千人千面 |
| **抖音** | 时长最大化 | 实时兴趣建模 | 沉浸式体验 |
| **网易云** | 付费转化 | 音乐理解 + 情感分析 | 每日推荐 |

| 平台 | 召回策略 | 排序模型 | 更新频率 |
|------|---------|---------|---------|
| **淘宝** | 多路召回（10+） | DeepFM+DIN | 分钟级 |
| **抖音** | 实时兴趣召回 | 多任务学习 | 秒级 |
| **网易云** | 协同过滤 + 内容 | 序列模型 | 小时级 |

## 2.8 本章小结

### 核心知识点

1. **电商推荐** — 全流程推荐，多目标优化
2. **视频推荐** — 时长最大化，实时反馈
3. **音乐推荐** — 兴趣稳定，场景化强
4. **新闻推荐** — 时效性极强，热点驱动
5. **社交推荐** — 关系图谱，隐私敏感
6. **本地生活** — LBS 推荐，低频高客单
