### 16.1.4 先做一个最小可运行版本

很多初学者一上来就想把召回、排序、重排、实时更新一次性做全，结果项目迟迟落不了地。更推荐按下面的顺序逐步推进：

| 阶段 | 目标 | 最小产出 |
|------|------|----------|

建议先把一个简单版本做对，再继续扩展深度模型或实时推荐。

## 16.3 推荐模型

### 16.3.1 协同过滤

```python
# models/cf.py
from surprise import KNNBasic, SVD
from surprise.model_selection import train_test_split
from surprise import accuracy
import pandas as pd

class CollaborativeFiltering:
    """协同过滤推荐"""
    
    def __init__(self, model_type='item'):
        """
        Args:
            model_type: 'user' 或 'item'
        """
        self.model_type = model_type
        
        if model_type == 'user':
            sim_options = {'name': 'pearson', 'user_based': True}
        else:
            sim_options = {'name': 'cosine', 'user_based': False}
        
        self.model = KNNBasic(
            sim_options=sim_options,
            k=50,
            min_k=5
        )
    
    def train(self, trainset):
        """训练模型"""
        self.model.fit(trainset)
        return self
    
    def predict(self, user_id, item_id):
        """预测评分"""
        return self.model.predict(user_id, item_id).est
    
    def recommend(self, user_id, all_items, n=10):
        """
        为用户推荐
        
        Args:
            user_id: 用户 ID
            all_items: 所有物品列表
            n: 推荐数量
        
        Returns:
            推荐列表 [(item_id, score), ...]
        """
        predictions = []
        
        for item_id in all_items:
            pred = self.predict(user_id, item_id)
            predictions.append((item_id, pred))
        
        predictions.sort(key=lambda x: x[1], reverse=True)
        return predictions[:n]


# 使用示例
if __name__ == "__main__":
    from surprise import Dataset
    
    data = Dataset.load_builtin('ml-100k')
    trainset, testset = train_test_split(data, test_size=0.25)
    
    model = CollaborativeFiltering(model_type='item')
    model.train(trainset)
    
    # 评估
    predictions = model.model.test(testset)
    print(f"RMSE: {accuracy.rmse(predictions):.4f}")
```

### 16.3.2 矩阵分解

```python
# models/mf.py
from surprise import SVD, NMF
from surprise.model_selection import cross_validate
import numpy as np

class MatrixFactorization:
    """矩阵分解推荐"""
    
    def __init__(
        self,
        n_factors=100,
        n_epochs=50,
        lr_all=0.005,
        reg_all=0.02
    ):
        """
        Args:
            n_factors: 隐含因子数量
            n_epochs: 训练轮数
            lr_all: 学习率
            reg_all: 正则化参数
        """
        self.model = SVD(
            n_factors=n_factors,
            n_epochs=n_epochs,
            lr_all=lr_all,
            reg_all=reg_all,
            random_state=42
        )
    
    def train(self, trainset):
        """训练模型"""
        self.model.fit(trainset)
        return self
    
    def predict(self, user_id, item_id):
        """预测评分"""
        return self.model.predict(user_id, item_id).est
    
    def get_user_factors(self, user_id):
        """获取用户隐含因子"""
        return self.model.pu[user_id]
    
    def get_item_factors(self, item_id):
        """获取物品隐含因子"""
        return self.model.qi[item_id]
    
    def similar_items(self, item_id, all_items, n=10):
        """
        查找相似物品
        
        Args:
            item_id: 物品 ID
            all_items: 所有物品
            n: 返回数量
        
        Returns:
            相似物品列表
        """
        from sklearn.metrics.pairwise import cosine_similarity
        
        item_vec = self.get_item_factors(item_id).reshape(1, -1)
        
        similarities = []
        for other_id in all_items:
            if other_id != item_id:
                other_vec = self.get_item_factors(other_id).reshape(1, -1)
                sim = cosine_similarity(item_vec, other_vec)[0][0]
                similarities.append((other_id, sim))
        
        similarities.sort(key=lambda x: x[1], reverse=True)
        return similarities[:n]


# 使用示例
if __name__ == "__main__":
    from surprise import Dataset, train_test_split
    from surprise import accuracy
    
    data = Dataset.load_builtin('ml-100k')
    trainset, testset = train_test_split(data, test_size=0.25)
    
    model = MatrixFactorization(n_factors=50, n_epochs=100)
    model.train(trainset)
    
    # 评估
    predictions = model.model.test(testset)
    print(f"RMSE: {accuracy.rmse(predictions):.4f}")
```

## 16.4 Web 应用

### 16.4.1 Flask 主程序

```python
# app.py
from flask import Flask, render_template, request, redirect, url_for
import pandas as pd
import sqlite3
from models.cf import CollaborativeFiltering
from models.mf import MatrixFactorization
from surprise import Dataset, Reader, Trainset

app = Flask(__name__)

# 全局变量
ratings_df = None
movies_df = None
model = None
trainset = None

def load_data():
    """加载数据"""
    global ratings_df, movies_df, model, trainset
    
    # 加载数据
    ratings_df = pd.read_csv('data/ml-latest-small/ratings.csv')
    movies_df = pd.read_csv('data/ml-latest-small/movies.csv')
    
    # 准备 Surprise 数据
    reader = Reader(rating_scale=(0.5, 5.0))
    data = Dataset.load_from_df(ratings_df[['userId', 'movieId', 'rating']], reader)
    trainset = data.build_full_trainset()
    
    # 训练模型
    model = MatrixFactorization(n_factors=50, n_epochs=100)
    model.train(trainset)
    
    print("数据加载完成！")


@app.route('/')
def index():
    """首页"""
    return render_template('index.html')


@app.route('/user')
def user_entry():
    """表单入口：把 query 参数转换为具体用户页"""
    user_id = request.args.get('user_id', type=int)
    if user_id is None:
        return redirect(url_for('index'))
    return redirect(url_for('user_page', user_id=user_id))


@app.route('/user/<int:user_id>')
def user_page(user_id):
    """用户页面"""
    # 获取用户评分过的电影
    user_ratings = ratings_df[ratings_df['userId'] == user_id]
    
    # 获取电影信息
    rated_movies = pd.merge(
        user_ratings,
        movies_df,
        on='movieId'
    )
    
    # 为用户推荐
    all_movies = movies_df['movieId'].tolist()
    recommendations = model.recommend(user_id, all_movies, n=10)
    
    # 获取推荐电影信息
    rec_movies = pd.merge(
        pd.DataFrame(recommendations, columns=['movieId', 'predicted_rating']),
        movies_df,
        on='movieId'
    )
    
    return render_template(
        'user.html',
        user_id=user_id,
        rated_movies=rated_movies,
        recommendations=rec_movies
    )


@app.route('/movie/<int:movie_id>')
def movie_page(movie_id):
    """电影页面"""
    movie = movies_df[movies_df['movieId'] == movie_id].iloc[0]
    
    # 获取相似电影
    all_movies = movies_df['movieId'].tolist()
    similar = model.similar_items(movie_id, all_movies, n=10)
    
    similar_movies = pd.merge(
        pd.DataFrame(similar, columns=['movieId', 'similarity']),
        movies_df,
        on='movieId'
    )
    
    return render_template(
        'movie.html',
        movie=movie,
        similar_movies=similar_movies
    )


if __name__ == '__main__':
    load_data()
    app.run(debug=True, host='0.0.0.0', port=5000)
```

### 16.5.5 先把离线评估面板做出来

很多项目最大的问题不是“模型不够复杂”，而是没有把结果展示清楚。一个最小可交付的离线评估面板，至少建议包含下面几项：

| 指标 | 作用 | 为什么要看 |
|------|------|--------------|
| RMSE | 衡量评分预测误差 | 适合看模型是否能大致拟合用户评分 |
| Precision@10 | 看前 10 个推荐里命中的比例 | 更贴近用户真正看到的推荐结果 |
| Recall@10 | 看用户喜欢的内容被找回多少 | 防止模型只推荐很少但看起来很准的结果 |
| Coverage | 看系统推荐是否过度集中在热门电影 | 判断推荐列表是否太单一 |

你可以把它做成最简单的一张表：

```text
模型         RMSE   Precision@10   Recall@10   Coverage
UserCF       0.982      0.214         0.163       18%
ItemCF       0.961      0.227         0.171       22%
SVD          0.913      0.241         0.186       27%
```

这张表的价值不只是“汇报结果”，更重要的是帮助你回答：

- 经典协同过滤和矩阵分解到底差在哪。
- 评分预测更准时，Top-N 推荐是否真的更好。
- 模型是不是过度偏向热门物品。

如果没有这一步，项目很容易停留在“能跑出推荐结果”，但很难说明为什么当前方案值得保留。

### 16.5.6 给推荐结果补解释信息

用户看到一串电影名时，往往会追问一句：“为什么推荐我这个？”

一个教学型项目不需要一次做到完整可解释 AI，但至少可以先补这 3 类解释：

1. 预测评分：告诉用户系统认为他可能会给这部电影打多少分。
2. 相似来源：告诉用户这个推荐和他历史上喜欢过的哪些电影更接近。
3. 类型线索：告诉用户这部电影和他偏好的类型是否一致。

可以把推荐结果组织成下面这种最小展示形式：

```text
Top 1: The Matrix
- predicted_rating: 4.62
- because_you_liked: Terminator 2, Star Wars
- matched_genres: Action, Sci-Fi
```

这样做有两个好处：

- 它能显著提升项目的可讲述性，面试或展示时更容易解释模型输出。
- 它能帮助你发现模型问题，比如推荐分很高，但类型明显不符合用户历史偏好。

如果你暂时做不到更复杂的解释，先把“预测评分 + 最近喜欢的相似电影 + 电影类型”拼出来，已经足够比纯 Top-N 列表更像一个完整项目。

如果你准备继续把这个项目打磨成作品集，建议优先按下面顺序升级：

所以更推荐你把最后的项目目标理解成：

> 不是做一个工业级推荐平台，而是做一个结构清晰、结果可展示、思路可复盘的推荐系统样例。

## 16.6 项目验收清单

做完本章后，至少应该能回答下面这些问题：

- 数据从哪里来，训练集和测试集怎么划分。
- 你用了什么模型，为什么先选这个模型。
- 你如何做离线评估，为什么不能只看 RMSE。
- 你的推荐接口输入和输出是什么。
- 如果用户很少评分，系统如何做冷启动处理。
- 如果要继续优化，下一步会优先改哪一层。

如果你想把它当成一个合格的作品集项目，还建议再检查下面这几项：

- README 里是否有项目结构图、运行方式和结果示例。
- 是否能展示至少一组真实推荐样例，而不只是代码截图。
- 是否能对比两个模型，而不只放一个默认实现。
- 是否能说明当前版本的局限，比如数据规模小、只做离线评估、没有在线反馈。

如果这些问题答不上来，说明项目可能只是“跑通了代码”，还没有真正变成你的作品。

## 16.8 常见坑

1. **把未评分当成负反馈。** MovieLens 中的 `0` 更适合作为缺失值，而不是明确不喜欢。
2. **只看一个指标。** RMSE 低不代表 Top-N 推荐效果一定好，最好同时看排序指标。
3. **样例能跑但项目讲不清。** 作品集项目除了代码，还需要明确的数据流、模型选择和迭代方向。
4. **一开始就追求大而全。** 先把离线评估和可演示页面做出来，再扩展复杂架构。
