## 项目定位

这个仓库不是单纯罗列论文，也不是简单堆一批代码模板，而是尽量回答 3 个更关键的问题：

1. 推荐系统到底在解决什么问题。
2. 常见算法为什么有效、各自适合什么场景。
3. 学完算法之后，怎样把它落到一个能讲清楚、能演示、能继续扩展的项目里。

目前内容已经覆盖 `19` 个章节，主要包括：

- 入门认知：推荐系统概念、应用场景、评估指标、环境搭建。
- 经典方法：协同过滤、基于内容推荐、矩阵分解、隐语义模型、冷启动。
- 深度推荐：NeuralCF、Wide&Deep、DeepFM、DIN/DIEN、多任务学习、序列推荐。
- 项目实战：电影推荐系统从数据、模型到 Web 交互的完整样例。
- 进阶方向：前沿技术、学习路线、继续进阶建议。

## 推荐学习方式

### 路线一：零基础入门

按 `01 -> 09 -> 16 -> 18` 的顺序读。

目标：先建立完整框架，再开始项目，不急着一上来读深度模型。

### 路线二：想尽快做项目

优先读 `01、03、05、07、09、16`。

目标：先把经典推荐系统搭起来，再决定是否深入 DeepFM、DIN、序列推荐。

### 路线三：准备面试或岗位转型

优先读 `03、05、07、09、11、12、13、14、16、19`。

目标：不仅会讲算法，还能讲评估、冷启动、项目拆解和迭代方向。

## 实战亮点

`16-电影推荐系统` 这一章不只是“给一个能跑的样例”，而是尽量引导你把它继续打磨成一个能展示、能复盘的推荐系统作品集项目。

这一部分重点强调 4 个交付物：

1. 离线评估结果：至少能对比 RMSE、Precision@10、Recall@10、Coverage。
2. 推荐结果展示：至少能输出用户 Top-N 推荐列表，而不是只训练模型。
3. 推荐解释信息：至少能说明预测评分、相似来源或匹配类型。
4. 项目复盘能力：能说清数据流、模型选择、评估方式和下一步迭代方向。

如果你是为了补项目经历、准备面试或做 GitHub 作品集，更建议优先把这一章做扎实，再继续补更复杂的模型。

## 快速开始

### 1. 克隆仓库

```bash
git clone https://github.com/xxuan66/recommender-system-tutorial.git
cd recommender-system-tutorial
```

### 2. 创建环境并安装依赖

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

说明：

- `scikit-surprise`、`implicit`、`lightfm` 主要用于经典推荐实验。
- `torch`、`tensorflow` 主要用于进阶模型示例。
- 如果你当前只阅读文档，可以先不跑全部代码；如果要做完整实验，建议使用独立虚拟环境。

### 3. 建议从哪里开始

如果你是第一次接触推荐系统，直接从 [01-什么是推荐系统](./01-入门篇/01-什么是推荐系统.md) 开始。

如果你已经有机器学习基础，更推荐从 [03-评估指标](./01-入门篇/03-评估指标.md) 和 [05-协同过滤算法](./02-基础篇/05-协同过滤算法.md) 切入。

## 这个仓库重点解决什么问题

### 1. 概念不再碎片化

很多推荐系统资料会把召回、排序、重排、冷启动、评估讲得很散。这里尽量把这些内容串成一条线，而不是一堆零散术语。

### 2. 算法不只停留在公式

每个阶段都尽量放进直观解释、适用场景、优缺点和代码实现，而不是只讲定义。

### 3. 学完后可以落到项目

实战篇重点不是“堆一个大而全的工业系统”，而是先给你一个足够清晰、可继续扩展的项目骨架，让你知道一个推荐项目应该如何拆。

## 推荐系统工具箱

| 工具 | 适用场景 | 说明 |
|------|----------|------|
| Surprise | 教学、评分预测、小规模实验 | 适合经典协同过滤和 SVD 入门 |
| Implicit | 隐式反馈推荐 | 适合 ALS、BPR 等场景 |
| LightFM | 混合推荐 | 适合冷启动和特征融合 |
| TensorFlow Recommenders | 深度推荐 | 适合 TensorFlow 生态 |
| RecBole | 快速实验 | 适合快速对比不同推荐模型 |

示例：

```python
from recbole.quick_start import run_recbole

run_recbole(
    model='BPR',
    dataset='ml-100k',
    config_dict={'epochs': 100}
)
```
