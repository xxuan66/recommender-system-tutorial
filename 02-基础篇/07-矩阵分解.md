# 第七章：矩阵分解

> 通过降维技术发现用户和物品的隐含特征

---

## 7.1 什么是矩阵分解

### 7.1.1 核心思想

**矩阵分解（Matrix Factorization）** 的基本思想是：

> 将用户 - 物品评分矩阵分解为两个低维矩阵的乘积，发现用户和物品的隐含特征。

### 7.1.2 数学表示

```
评分矩阵 R ≈ P × Q^T

其中：
- R: m×n 评分矩阵（m 个用户，n 个物品）
- P: m×k 用户隐含特征矩阵
- Q: n×k 物品隐含特征矩阵
- k: 隐含特征维度（k << m, k << n）
```

**图示：**
```
R (m×n)     ≈     P (m×k)    ×    Q^T (k×n)

[用户 1]  [5  3  0  4]           [0.5  0.3]    [0.8  0.2  0.1  0.9]
[用户 2]  [4  0  3  5]     ≈     [0.3  0.7]  × [0.6  0.8  0.4  0.5]
[用户 3]  [0  4  5  0]           [0.9  0.1]    [0.2  0.9  0.7  0.3]
                                ↑             ↑
                            用户特征       物品特征
                            (动作/爱情)   (动作/爱情)
```

### 7.1.3 为什么有效

**隐含特征的例子（电影推荐）：**

| 隐含特征 1 | 隐含特征 2 | 隐含特征 3 |
|-----------|-----------|-----------|
| 动作程度 | 爱情程度 | 科幻程度 |
| 喜剧程度 | 剧情程度 | 惊悚程度 |
| 商业程度 | 艺术程度 | 经典程度 |

**用户和物品都可以用这些特征表示：**
- 用户 A：[0.8, 0.2, 0.6] — 喜欢动作、科幻
- 电影 X：[0.9, 0.1, 0.8] — 动作科幻片
- 预测评分：0.8×0.9 + 0.2×0.1 + 0.6×0.8 = 1.22

### 7.1.4 它比协同过滤进了一步在哪里

前一章的协同过滤，本质上还是在直接比较“用户和用户像不像”或“物品和物品像不像”。

矩阵分解往前走了一步：

1. 不再只看显式的相似度。
2. 尝试把用户兴趣和物品属性压缩到一个更低维的隐空间里。
3. 用这个隐空间去解释“为什么某个用户会喜欢某个物品”。

这也是矩阵分解在推荐系统里长期重要的原因：

- 它能在稀疏评分矩阵中提取更稳定的结构。
- 它比纯相似度方法更容易泛化到未见过的用户 - 物品组合。
- 它为后续更复杂的推荐模型提供了统一视角：用户向量、物品向量、交互函数。

## 7.2 SVD（奇异值分解）

### 7.2.1 原理

**SVD 分解公式：**
```
R = U × Σ × V^T

其中：
- U: m×m 正交矩阵（左奇异向量）
- Σ: m×n 对角矩阵（奇异值）
- V: n×n 正交矩阵（右奇异向量）
```

**降维近似：**
```
R ≈ U_k × Σ_k × V_k^T

其中 k 是前 k 个最大的奇异值
```

### 7.2.2 代码实现

```python
import numpy as np
from scipy.sparse.linalg import svds

class SVDRecommender:
    """基于 SVD 的推荐系统"""
    
    def __init__(self, k=10):
        """
        Args:
            k: 隐含特征维度
        """
        self.k = k
        
    def fit(self, ratings):
        """
        训练模型
        
        Args:
            ratings: 用户 - 物品评分矩阵
        """
        self.ratings = ratings
        
        # 填充缺失值（用全局平均分）
        self.global_mean = np.mean(ratings[ratings > 0])
        ratings_filled = ratings.copy()
        ratings_filled[ratings == 0] = self.global_mean
        
        # SVD 分解
        U, sigma, Vt = svds(ratings_filled, k=self.k)
        
        # 保存分解结果
        self.U = U
        self.sigma = np.diag(sigma)
        self.Vt = Vt
        
        return self
    
    def predict(self, u, i):
        """预测评分"""
        pred = np.dot(np.dot(self.U[u], self.sigma), self.Vt[:, i])
        return pred
    
    def recommend(self, u, n=10):
        """推荐 Top-N 物品"""
        n_items = self.ratings.shape[1]
        
        # 预测所有物品评分
        predictions = []
        for i in range(n_items):
            if self.ratings[u, i] == 0:  # 未评分
                score = self.predict(u, i)
                predictions.append((i, score))
        
        # 排序
        predictions.sort(key=lambda x: x[1], reverse=True)
        return predictions[:n]


# 使用示例
if __name__ == "__main__":
    ratings = np.array([
        [5, 3, 0, 4, 0],
        [4, 0, 3, 5, 2],
        [0, 4, 5, 0, 3],
        [3, 0, 4, 0, 5],
        [0, 5, 0, 3, 4],
    ])
    
    model = SVDRecommender(k=3)
    model.fit(ratings)
    
    user_id = 0
    recommendations = model.recommend(user_id, n=3)
    
    print(f"为用户 {user_id} 推荐的物品：")
    for item_id, score in recommendations:
        print(f"  物品 {item_id}: 预测评分 {score:.2f}")
```

### 7.2.3 SVD 的局限性

| 问题 | 说明 |
|------|------|
| **需要稠密矩阵** | SVD 要求矩阵是稠密的，需要填充缺失值 |
| **无法处理稀疏数据** | 填充会引入噪声 |
| **计算复杂度高** | O(m×n×k)，大数据集慢 |
| **无法增量更新** | 新数据需要重新分解 |

## 7.3  FunkSVD

### 7.3.1 改进思路

**FunkSVD（Simon Funk 提出）** 的核心改进：

> 不填充缺失值，只优化已知的评分

### 7.3.2 优化目标

```
最小化：Σ(u,i)∈R (ru,i - pu^T × qi)^2 + λ(||pu||^2 + ||qi||^2)

其中：
- ru,i: 用户 u 对物品 i 的真实评分
- pu: 用户 u 的隐含特征向量
- qi: 物品 i 的隐含特征向量
- λ: 正则化参数（防止过拟合）
```

### 7.3.3 梯度下降优化

```python
class FunkSVD:
    """FunkSVD 实现"""
    
    def __init__(self, k=10, lr=0.01, reg=0.02, epochs=100):
        """
        Args:
            k: 隐含特征维度
            lr: 学习率
            reg: 正则化参数
            epochs: 训练轮数
        """
        self.k = k
        self.lr = lr
        self.reg = reg
        self.epochs = epochs
        
    def fit(self, ratings):
        """训练模型"""
        self.ratings = ratings
        n_users, n_items = ratings.shape
        
        # 随机初始化 P 和 Q
        np.random.seed(42)
        self.P = np.random.normal(0, 0.1, (n_users, self.k))
        self.Q = np.random.normal(0, 0.1, (n_items, self.k))
        
        # 全局偏置和用户/物品偏置
        self.global_mean = np.mean(ratings[ratings > 0])
        self.bu = np.zeros(n_users)  # 用户偏置
        self.bi = np.zeros(n_items)  # 物品偏置
        
        # 找到所有已知评分
        known_ratings = []
        for u in range(n_users):
            for i in range(n_items):
                if ratings[u, i] > 0:
                    known_ratings.append((u, i, ratings[u, i]))
        
        # 梯度下降
        for epoch in range(self.epochs):
            np.random.shuffle(known_ratings)
            total_error = 0
            
            for u, i, r in known_ratings:
                # 预测
                pred = self.global_mean + self.bu[u] + self.bi[i] + np.dot(self.P[u], self.Q[i])
                
                # 误差
                error = r - pred
                total_error += error ** 2
                
                # 更新偏置
                self.bu[u] += self.lr * (error - self.reg * self.bu[u])
                self.bi[i] += self.lr * (error - self.reg * self.bi[i])
                
                # 更新隐含特征
                pu_old = self.P[u].copy()
                self.P[u] += self.lr * (error * self.Q[i] - self.reg * self.P[u])
                self.Q[i] += self.lr * (error * pu_old - self.reg * self.Q[i])
            
            # 打印训练进度
            if (epoch + 1) % 20 == 0:
                rmse = np.sqrt(total_error / len(known_ratings))
                print(f"Epoch {epoch+1}/{self.epochs}, RMSE: {rmse:.4f}")
        
        return self
    
    def predict(self, u, i):
        """预测评分"""
        pred = self.global_mean + self.bu[u] + self.bi[i] + np.dot(self.P[u], self.Q[i])
        return np.clip(pred, 0.5, 5.0)  # 限制在评分范围内
    
    def recommend(self, u, n=10):
        """推荐 Top-N 物品"""
        n_items = self.ratings.shape[1]
        
        predictions = []
        for i in range(n_items):
            if self.ratings[u, i] == 0:
                score = self.predict(u, i)
                predictions.append((i, score))
        
        predictions.sort(key=lambda x: x[1], reverse=True)
        return predictions[:n]


# 使用示例
if __name__ == "__main__":
    ratings = np.array([
        [5, 3, 0, 4, 0],
        [4, 0, 3, 5, 2],
        [0, 4, 5, 0, 3],
        [3, 0, 4, 0, 5],
        [0, 5, 0, 3, 4],
    ])
    
    model = FunkSVD(k=5, lr=0.01, reg=0.02, epochs=100)
    model.fit(ratings)
    
    user_id = 0
    recommendations = model.recommend(user_id, n=3)
    
    print(f"\n为用户 {user_id} 推荐的物品：")
    for item_id, score in recommendations:
        print(f"  物品 {item_id}: 预测评分 {score:.2f}")
```

## 7.4 SVD++

### 7.4.1 改进思路

**SVD++** 在 FunkSVD 基础上增加了：

> 考虑用户的隐式反馈（浏览、点击等行为）

### 7.4.2 预测公式

```
P(u, i) = μ + bu + bi + qi^T × (pu + |Iu|^(-0.5) × Σ(j∈Iu) yj)

其中：
- Iu: 用户 u 有过隐式反馈的物品集合
- yj: 物品 j 的隐式反馈向量
- |Iu|^(-0.5): 归一化因子
```

### 7.4.3 代码实现

```python
class SVDPlusPlus:
    """SVD++ 实现"""
    
    def __init__(self, k=10, lr=0.01, reg=0.02, epochs=100):
        self.k = k
        self.lr = lr
        self.reg = reg
        self.epochs = epochs
        
    def fit(self, ratings, implicit_feedback=None):
        """
        训练模型
        
        Args:
            ratings: 显式评分矩阵
            implicit_feedback: 隐式反馈矩阵（0/1）
        """
        self.ratings = ratings
        n_users, n_items = ratings.shape
        
        # 隐式反馈（如果没有，用评分矩阵代替）
        if implicit_feedback is None:
            self.implicit = (ratings > 0).astype(float)
        else:
            self.implicit = implicit_feedback
        
        # 初始化参数
        np.random.seed(42)
        self.P = np.random.normal(0, 0.1, (n_users, self.k))
        self.Q = np.random.normal(0, 0.1, (n_items, self.k))
        self.Y = np.random.normal(0, 0.1, (n_items, self.k))  # 隐式反馈向量
        
        self.global_mean = np.mean(ratings[ratings > 0])
        self.bu = np.zeros(n_users)
        self.bi = np.zeros(n_items)
        
        # 找到已知评分
        known_ratings = []
        for u in range(n_users):
            for i in range(n_items):
                if ratings[u, i] > 0:
                    known_ratings.append((u, i, ratings[u, i]))
        
        # 梯度下降
        for epoch in range(self.epochs):
            np.random.shuffle(known_ratings)
            total_error = 0
            
            for u, i, r in known_ratings:
                # 用户隐式反馈集合
                Iu = np.where(self.implicit[u] > 0)[0]
                
                # 隐式反馈的平均向量
                if len(Iu) > 0:
                    y_u = np.mean(self.Y[Iu], axis=0)
                else:
                    y_u = np.zeros(self.k)
                
                # 预测
                pred = self.global_mean + self.bu[u] + self.bi[i] + \
                       np.dot(self.Q[i], self.P[u] + y_u)
                
                error = r - pred
                total_error += error ** 2
                
                # 更新参数
                self.bu[u] += self.lr * (error - self.reg * self.bu[u])
                self.bi[i] += self.lr * (error - self.reg * self.bi[i])
                
                pu_old = self.P[u].copy()
                self.P[u] += self.lr * (error * self.Q[i] - self.reg * self.P[u])
                self.Q[i] += self.lr * (error * (pu_old + y_u) - self.reg * self.Q[i])
                
                # 更新隐式反馈向量
                if len(Iu) > 0:
                    for j in Iu:
                        self.Y[j] += self.lr * (error * self.Q[i] / len(Iu) - self.reg * self.Y[j])
            
            if (epoch + 1) % 20 == 0:
                rmse = np.sqrt(total_error / len(known_ratings))
                print(f"Epoch {epoch+1}/{self.epochs}, RMSE: {rmse:.4f}")
        
        return self
    
    def predict(self, u, i):
        """预测评分"""
        Iu = np.where(self.implicit[u] > 0)[0]
        
        if len(Iu) > 0:
            y_u = np.mean(self.Y[Iu], axis=0)
        else:
            y_u = np.zeros(self.k)
        
        pred = self.global_mean + self.bu[u] + self.bi[i] + \
               np.dot(self.Q[i], self.P[u] + y_u)
        
        return np.clip(pred, 0.5, 5.0)
    
    def recommend(self, u, n=10):
        """推荐 Top-N 物品"""
        n_items = self.ratings.shape[1]
        
        predictions = []
        for i in range(n_items):
            if self.ratings[u, i] == 0:
                score = self.predict(u, i)
                predictions.append((i, score))
        
        predictions.sort(key=lambda x: x[1], reverse=True)
        return predictions[:n]
```

## 7.6 如何选择 SVD、FunkSVD 和 SVD++

这一节是很多读者最容易混淆的地方。名字都很像，但适用场景并不完全一样。

### 7.6.1 一个快速判断表

| 方法 | 更适合的情况 | 不足 |
|------|--------------|------|
| SVD | 教学、理解分解思想、小规模实验 | 需要先处理缺失值，对稀疏数据不够自然 |
| FunkSVD | 显式评分推荐、较常规的矩阵分解实验 | 需要调较多超参数 |
| SVD++ | 既有显式反馈，也想利用隐式行为 | 训练更慢，复杂度更高 |

### 7.6.2 实际使用建议

1. 如果你是第一次学矩阵分解，先理解 SVD 的分解视角。
2. 如果你要在推荐任务里真正做实验，通常更该重点理解 FunkSVD。
3. 如果你的数据里除了评分还有点击、浏览、收藏等行为，再看 SVD++ 会更有意义。

一个常见误区是：

> 以为更复杂的方法一定更好。

实际上，越复杂的方法通常越依赖更多数据、更细致的调参和更高的训练成本。

## 7.7 参数怎么调更有感觉

初学者经常会把矩阵分解的参数当成黑箱。下面这几个参数最值得先理解：

### 7.7.1 隐含特征维度 `k`

- `k` 太小：表达能力不足，模型容易欠拟合。
- `k` 太大：模型容易记住噪声，训练更慢，也更容易过拟合。

经验上：

- 教学实验可以从 `20`、`50`、`100` 这样的量级开始。
- 不要一开始就把 `k` 调得很大，先看效果趋势更重要。

### 7.7.2 学习率 `lr`

- 太小：收敛慢，看起来像“怎么都学不动”。
- 太大：训练震荡，甚至直接发散。

建议：

- 先从 `0.005` 或 `0.01` 这类常见值开始。
- 观察训练过程中的 RMSE 是否稳定下降。

### 7.7.3 正则化 `reg`

- 太小：更容易过拟合。
- 太大：模型会被约束得过强，表达能力下降。

建议：

- 不要孤立地调 `reg`，要和 `k`、`lr` 一起看。
- 参数调优的目标不是某个参数“理论最好”，而是验证集效果更稳。

### 7.7.4 训练轮数 `epochs`

- 轮数太少：模型还没学够。
- 轮数太多：可能继续压训练误差，但验证集不再提升。

建议：

- 每次记录训练集和验证集效果。
- 如果验证集效果不再改善，就要考虑早停或减少训练轮数。

---

### 7.7.5 一个推荐的调参顺序

初学者调矩阵分解时，最容易出现的问题不是“不会调”，而是一次同时改太多参数，最后完全不知道结果为什么变化。

更推荐按下面顺序来：

1. 先固定评估方式和数据划分。
2. 先调 `k`，看模型表达能力的变化。
3. 再调 `reg`，看是否有明显过拟合。
4. 最后调 `lr` 和 `epochs`，让训练过程更稳定。

这样做的好处是：

- 你更容易看出每个参数在改变什么。
- 你能避免“分数变了，但不知道为什么”的黑箱感。
- 你后面写实验总结时，也更容易把过程讲清楚。

## 7.8 矩阵分解对比

| 算法 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **SVD** | 简单、理论基础强 | 需要稠密矩阵 | 小规模数据 |
| **FunkSVD** | 处理稀疏数据 | 需要调参 | 一般推荐 |
| **SVD++** | 利用隐式反馈 | 计算复杂 | 有隐式反馈数据 |

---

### 7.8.1 矩阵分解比协同过滤多解决了什么

这是这一章最关键的问题之一。

协同过滤主要依赖直接相似度，而矩阵分解进一步尝试学习用户和物品的隐向量表示，因此它通常在下面几方面更有优势：

| 维度 | 协同过滤 | 矩阵分解 |
|------|----------|----------|
| 核心思路 | 直接找相似用户 / 相似物品 | 学习用户和物品的隐空间表示 |
| 对稀疏数据的处理 | 容易受共同交互不足影响 | 更容易从整体结构中提取模式 |
| 可扩展性 | 大规模相似度计算成本较高 | 更容易形成统一的向量化表示 |
| 后续扩展 | 更适合做 baseline | 更容易衔接深度推荐模型 |

但这不代表矩阵分解一定全面优于协同过滤。

如果你的目标是：

- 快速做一个可解释 baseline
- 在小规模数据上验证流程
- 向读者讲清推荐系统最基本的推荐逻辑

那么协同过滤依然很有价值。

更准确的理解应该是：

> 协同过滤适合先立住基线，矩阵分解适合继续提升表达能力和泛化能力。

## 7.9 最小实验建议

如果你准备把这章内容转成一个小实验，可以按下面顺序做：

1. 用 MovieLens 跑一个简单 baseline。
2. 固定评估方式，只改一个参数观察变化。
3. 对比 `k`、`reg`、`epochs` 对效果的影响。
4. 再拿协同过滤做一组简单对比。

最重要的不是把分数刷到最好，而是弄清楚：

- 你的数据更适合哪类方法。
- 不同参数为什么会影响结果。
- 矩阵分解相对协同过滤到底解决了什么问题。

### 7.9.1 怎样算一个合格的矩阵分解实验

如果你准备把这章内容写进课程作业、博客或作品集，建议至少补齐下面这些交付物：

- 一张参数对比表：记录 `k`、`reg`、`epochs` 的变化和结果。
- 一组模型对比：至少拿 `FunkSVD` 和协同过滤 baseline 做对比。
- 一段结果解释：说明为什么当前参数或模型更合适。
- 一段局限性说明：例如只做了显式评分、没有使用上下文特征、没有在线验证。

这样做的价值在于，你不只是展示“我跑过矩阵分解”，而是在展示：

1. 你知道怎么设实验。
2. 你知道怎么解释结果。
3. 你知道当前方法还缺什么。

这比单独贴一张 RMSE 数字更能体现你对推荐系统的理解。

## 7.10 常见误区

### 7.10.1 以为隐向量有唯一的现实语义

隐向量维度有时可以解释成“动作偏好”“爱情偏好”，但这更多是帮助理解的直觉，并不意味着每一维都有唯一且稳定的人类语义。

### 7.10.2 只要分解了矩阵，就算理解了推荐系统

矩阵分解很重要，但推荐系统不只有建模。样本构造、评估指标、冷启动、工程部署同样关键。

### 7.10.3 只盯着离线误差

和协同过滤类似，矩阵分解离线误差更低，并不自动意味着线上推荐体验一定更好。

### 7.10.4 忽略数据稀疏和反馈类型

如果你的数据主要是点击、浏览、购买，而不是明确评分，就要非常注意显式反馈方法和隐式反馈场景之间的差异。
