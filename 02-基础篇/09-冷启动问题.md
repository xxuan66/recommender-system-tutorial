# 第九章：冷启动问题

> 推荐系统绕不开的核心问题之一：没有历史数据时，系统还能怎么推荐

---

## 9.1 什么是冷启动

### 9.1.1 定义

**冷启动（Cold Start）** 可以先定义为：

> 推荐系统无法为新用户或新物品提供个性化推荐的问题。

### 9.1.2 冷启动的类型

```
冷启动问题
├── 用户冷启动
│   └── 新用户没有历史行为数据
├── 物品冷启动
│   └── 新物品没有被用户交互过
└── 系统冷启动
    └── 新平台既没有用户数据也没有物品数据
```

### 9.1.3 冷启动的影响

| 影响 | 说明 |
|------|------|
| **用户体验差** | 推荐不相关，用户流失 |
| **平台损失** | 新物品无法获得曝光 |
| **马太效应** | 热门更热，冷门更冷 |
| **生态恶化** | 内容创作者失去动力 |

### 9.1.4 为什么冷启动不是一个“小问题”

很多人初学推荐系统时，会把冷启动当成一个边角问题，觉得等模型做得更强自然就能解决。实际上，冷启动往往是推荐系统最先暴露出来的问题之一。

原因很简单：

1. 再强的协同过滤或矩阵分解，也需要历史交互数据。
2. 新用户第一次进入系统时，恰恰最缺的就是历史数据。
3. 新物品刚上线时，如果不给机会，永远也积累不到交互。

所以冷启动不是“等系统成熟了再看”的问题，而是推荐系统从第一天就必须面对的问题。

## 9.2 用户冷启动

### 9.2.1 原因分析

**新用户没有历史行为，无法：**
- 计算用户相似度（UserCF）
- 构建用户画像（基于内容）
- 学习用户隐含因子（LFM）

### 9.2.2 解决方案

#### 方案 1：热门推荐

**最简单的方案：推荐热门物品**

```python
class PopularRecommender:
    """热门推荐"""
    
    def fit(self, ratings):
        """统计物品热度"""
        self.ratings = ratings
        
        # 统计每个物品的交互次数
        item_popularity = np.sum(ratings > 0, axis=0)
        
        # 按热度排序
        self.item_ranking = np.argsort(item_popularity)[::-1]
        
        return self
    
    def recommend(self, user_id, n=10):
        """返回热门物品"""
        return [(item_id, 0) for item_id in self.item_ranking[:n]]


# 使用
model = PopularRecommender()
model.fit(ratings)
recommendations = model.recommend(user_id=None, n=10)
```

**优点：**
- 简单有效
- 无需用户数据
- 热门物品质量通常较高

**缺点：**
- 缺乏个性化
- 马太效应严重
- 新物品无法曝光

#### 方案 2：基于人口统计

**利用用户注册信息：**

```python
class DemographicRecommender:
    """基于人口统计的推荐"""
    
    def __init__(self):
        self.user_profiles = {}
        self.group_preferences = {}
    
    def add_user_info(self, user_id, age, gender, location, interests):
        """添加用户信息"""
        self.user_profiles[user_id] = {
            'age': age,
            'gender': gender,
            'location': location,
            'interests': interests,
        }
    
    def fit(self, ratings, user_ids):
        """学习群体偏好"""
        # 按年龄段分组
        age_groups = {'young': [], 'middle': [], 'old': []}
        for i, user_id in enumerate(user_ids):
            if user_id in self.user_profiles:
                age = self.user_profiles[user_id]['age']
                if age < 25:
                    age_groups['young'].append(i)
                elif age < 45:
                    age_groups['middle'].append(i)
                else:
                    age_groups['old'].append(i)
        
        # 计算各群体偏好
        for group, indices in age_groups.items():
            if len(indices) > 0:
                group_ratings = ratings[indices]
                self.group_preferences[group] = np.mean(group_ratings, axis=0)
        
        return self
    
    def recommend(self, user_id, n=10):
        """基于用户群体推荐"""
        if user_id not in self.user_profiles:
            return []
        
        # 确定年龄段
        age = self.user_profiles[user_id]['age']
        if age < 25:
            group = 'young'
        elif age < 45:
            group = 'middle'
        else:
            group = 'old'
        
        if group not in self.group_preferences:
            return []
        
        # 推荐该群体喜欢的物品
        preferences = self.group_preferences[group]
        top_items = np.argsort(preferences)[::-1][:n]
        
        return [(item_id, preferences[item_id]) for item_id in top_items]
```

#### 方案 3：主动询问

**让用户选择感兴趣的类别：**

```python
class OnboardingRecommender:
    """新用户引导推荐"""
    
    def __init__(self):
        self.user_interests = {}
        self.item_categories = {}
    
    def set_user_interests(self, user_id, categories):
        """
        设置用户兴趣类别
        
        Args:
            user_id: 用户 ID
            categories: 用户选择的类别列表
        """
        self.user_interests[user_id] = categories
    
    def add_item(self, item_id, category):
        """添加物品类别"""
        self.item_categories[item_id] = category
    
    def recommend(self, user_id, n=10):
        """基于兴趣类别推荐"""
        if user_id not in self.user_interests:
            return []
        
        interests = self.user_interests[user_id]
        
        # 找出感兴趣类别的物品
        candidate_items = []
        for item_id, category in self.item_categories.items():
            if category in interests:
                candidate_items.append(item_id)
        
        # 随机推荐（或按其他规则排序）
        import random
        random.shuffle(candidate_items)
        
        return [(item_id, 0) for item_id in candidate_items[:n]]


# 使用示例
recommender = OnboardingRecommender()

# 新用户注册时选择兴趣

# 添加物品类别

recommender.add_item("movie_2", "爱情")
recommender.add_item("movie_3", "动作")
recommender.add_item("news_1", "科技")
```

#### 方案 4：探索与利用

**Epsilon-Greedy 算法：**

```python
import numpy as np

class EpsilonGreedyRecommender:
    """Epsilon-Greedy 探索利用推荐"""
    
    def __init__(self, epsilon=0.1):
        """
        Args:
            epsilon: 探索概率（0-1）
        """
        self.epsilon = epsilon
        self.item_rewards = {}  # 物品累计奖励
        self.item_counts = {}   # 物品被推荐次数
    
    def recommend(self, all_items, n=10):
        """
        推荐物品
        
        Args:
            all_items: 所有物品列表
            n: 推荐数量
        """
        recommended = []
        
        for _ in range(n):
            # 以 epsilon 概率探索
            if np.random.random() < self.epsilon:
                # 探索：随机选择
                available = [item for item in all_items if item not in recommended]
                if available:
                    item = np.random.choice(available)
                else:
                    break
            else:
                # 利用：选择平均奖励最高的
                if not self.item_counts:
                    # 没有数据时随机
                    available = [item for item in all_items if item not in recommended]
                    if available:
                        item = np.random.choice(available)
                    else:
                        break
                else:
                    # 计算平均奖励
                    avg_rewards = {
                        item: self.item_rewards.get(item, 0) / max(self.item_counts.get(item, 1), 1)
                        for item in all_items if item not in recommended
                    }
                    item = max(avg_rewards, key=avg_rewards.get)
            
            recommended.append(item)
        
        return [(item, 0) for item in recommended]
    
    def update(self, item, reward):
        """
        更新物品奖励
        
        Args:
            item: 物品 ID
            reward: 奖励（点击=1，购买=2，收藏=3）
        """
        self.item_rewards[item] = self.item_rewards.get(item, 0) + reward
        self.item_counts[item] = self.item_counts.get(item, 0) + 1


# 使用示例
recommender = EpsilonGreedyRecommender(epsilon=0.3)

all_items = list(range(100))

# 第 1 次推荐（探索为主）
recs = recommender.recommend(all_items, n=5)
print(f"第 1 次推荐：{recs}")

# 用户反馈
recommender.update(recs[0][0], reward=1)  # 点击
recommender.update(recs[1][0], reward=0)  # 未点击

# 第 2 次推荐（开始利用）
recs = recommender.recommend(all_items, n=5)
print(f"第 2 次推荐：{recs}")
```

## 9.3 物品冷启动

### 9.3.1 原因分析

**新物品没有被用户交互过，无法：**
- 计算物品相似度（ItemCF）
- 学习物品隐含因子（LFM）
- 进入推荐候选集

### 9.3.2 解决方案

#### 方案 1：基于内容推荐

**利用物品内容特征：**

```python
class ContentBasedColdStart:
    """基于内容的冷启动推荐"""
    
    def __init__(self):
        self.item_features = {}
        self.user_profiles = {}
    
    def add_item(self, item_id, features):
        """
        添加新物品
        
        Args:
            item_id: 物品 ID
            features: 物品特征向量
        """
        self.item_features[item_id] = features
    
    def add_user_interaction(self, user_id, item_id, rating=1.0):
        """添加用户交互"""
        if user_id not in self.user_profiles:
            self.user_profiles[user_id] = []
        
        if item_id in self.item_features:
            self.user_profiles[user_id].append({
                'item_id': item_id,
                'features': self.item_features[item_id],
                'rating': rating,
            })
    
    def recommend_new_item(self, item_id, user_id):
        """
        判断是否向用户推荐新物品
        
        Args:
            item_id: 新物品 ID
            user_id: 用户 ID
        
        Returns:
            推荐分数
        """
        if user_id not in self.user_profiles:
            return 0.5  # 中立分数
        
        if item_id not in self.item_features:
            return 0
        
        new_item_features = self.item_features[item_id]
        
        # 计算与用户历史喜欢物品的相似度
        from sklearn.metrics.pairwise import cosine_similarity
        
        user_history = self.user_profiles[user_id]
        liked_items = [item for item in user_history if item['rating'] >= 4]
        
        if len(liked_items) == 0:
            return 0.5
        
        # 平均相似度
        similarities = []
        for item in liked_items:
            sim = cosine_similarity(
                [new_item_features],
                [item['features']]
            )[0][0]
            similarities.append(sim)
        
        return np.mean(similarities)
```

#### 方案 2：流量扶持

**给新物品一定的曝光机会：**

```python
class BoostNewItems:
    """新物品流量扶持"""
    
    def __init__(self, boost_days=7, boost_factor=0.3):
        """
        Args:
            boost_days: 扶持天数
            boost_factor: 扶持系数（0-1）
        """
        self.boost_days = boost_days
        self.boost_factor = boost_factor
        self.item_create_time = {}
    
    def add_item(self, item_id, create_time=None):
        """添加物品"""
        import time
        if create_time is None:
            create_time = time.time()
        self.item_create_time[item_id] = create_time
    
    def get_boost_score(self, item_id):
        """获取扶持分数"""
        import time
        
        if item_id not in self.item_create_time:
            return 0
        
        create_time = self.item_create_time[item_id]
        now = time.time()
        
        # 计算天数
        days_elapsed = (now - create_time) / (24 * 3600)
        
        if days_elapsed > self.boost_days:
            return 0
        
        # 线性衰减
        boost = self.boost_factor * (1 - days_elapsed / self.boost_days)
        return boost
    
    def rerank_with_boost(self, recommendations):
        """
        重新排序（加入扶持分数）
        
        Args:
            recommendations: 原始推荐列表 [(item_id, score), ...]
        
        Returns:
            重新排序后的列表
        """
        boosted = []
        for item_id, score in recommendations:
            boost = self.get_boost_score(item_id)
            boosted_score = score + boost
            boosted.append((item_id, boosted_score))
        
        # 重新排序
        boosted.sort(key=lambda x: x[1], reverse=True)
        return boosted
```

#### 方案 3：Bandit 算法

**Thompson Sampling：**

```python
import numpy as np

class ThompsonSampling:
    """Thompson Sampling 用于新物品探索"""
    
    def __init__(self):
        # 每个物品的成功/失败次数（Beta 分布参数）
        self.successes = {}
        self.failures = {}
    
    def select_items(self, all_items, n=10):
        """
        选择物品
        
        Args:
            all_items: 所有物品
            n: 选择数量
        """
        sampled_scores = {}
        
        for item in all_items:
            # 从 Beta 分布采样
            alpha = self.successes.get(item, 1)
            beta = self.failures.get(item, 1)
            sampled_scores[item] = np.random.beta(alpha, beta)
        
        # 选择分数最高的 n 个
        sorted_items = sorted(sampled_scores.items(), key=lambda x: x[1], reverse=True)
        return [(item, score) for item, score in sorted_items[:n]]
    
    def update(self, item, success):
        """
        更新物品统计
        
        Args:
            item: 物品 ID
            success: 是否成功（点击/购买）
        """
        if success:
            self.successes[item] = self.successes.get(item, 0) + 1
        else:
            self.failures[item] = self.failures.get(item, 0) + 1
```

```python
class RuleBasedRecommender:
    """基于规则的推荐（冷启动期）"""
    
    def __init__(self):
        self.rules = []
    
    def add_rule(self, rule_func, weight=1.0):
        """
        添加规则
        
        Args:
            rule_func: 规则函数 (user, items) -> scores
            weight: 规则权重
        """
        self.rules.append((rule_func, weight))
    
    def recommend(self, user, items, n=10):
        """基于规则推荐"""
        if not self.rules:
            # 没有规则时随机推荐
            import random
            random.shuffle(items)
            return items[:n]
        
        # 综合所有规则
        scores = {item: 0 for item in items}
        
        for rule_func, weight in self.rules:
            rule_scores = rule_func(user, items)
            for item, score in rule_scores.items():
                scores[item] += score * weight
        
        # 排序
        sorted_items = sorted(scores.items(), key=lambda x: x[1], reverse=True)
        return [item for item, score in sorted_items[:n]]


# 规则示例
def diversity_rule(user, items):
    """多样性规则：确保推荐不同类别"""
    scores = {}
    categories = set()
    for item in items:
        if item['category'] not in categories:
            scores[item['id']] = 1
            categories.add(item['category'])
        else:
            scores[item['id']] = 0.5
    return scores

def quality_rule(user, items):
    """质量规则：优先推荐高质量内容"""
    return {item['id']: item['quality_score'] for item in items}

# 使用
recommender = RuleBasedRecommender()
recommender.add_rule(diversity_rule, weight=0.4)
recommender.add_rule(quality_rule, weight=0.6)

recommendations = recommender.recommend(user, items, n=10)
```

## 9.5 混合策略

### 9.5.1 多策略融合

```python
class HybridColdStart:
    """混合冷启动策略"""
    
    def __init__(self):
        self.popular_recommender = PopularRecommender()
        self.content_recommender = ContentBasedColdStart()
        self.bandit = ThompsonSampling()
    
    def recommend(self, user_id, items, n=10, user_type='new'):
        """
        混合推荐
        
        Args:
            user_id: 用户 ID
            items: 候选物品
            n: 推荐数量
            user_type: 用户类型 ('new', 'returning')
        """
        if user_type == 'new':
            # 新用户：热门 + 探索
            popular = self.popular_recommender.recommend(user_id, n // 2)
            explore = self.bandit.select_items(items, n // 2)
            
            # 融合
            combined = popular + explore
            return combined[:n]
        else:
            # 回归用户：基于内容
            return self.content_recommender.recommend_new_item(items[0], user_id)
```

### 9.5.2 阶段性策略

```
冷启动阶段演进：

初期（0-1000 用户）
└→ 人工运营 + 规则推荐

成长期（1000-10000 用户）
└→ 热门推荐 + 探索算法

成熟期（10000+ 用户）
└→ 个性化推荐 + 冷启动补充
```

## 9.6 一个实用的决策框架

学完前面的方案后，更重要的是知道实际遇到问题时怎么选，而不是把所有方案都堆上去。

### 9.6.1 如果是新用户冷启动

优先考虑顺序：

1. 热门内容或高质量内容兜底。
2. Onboarding 主动收集兴趣。
3. 使用少量探索策略快速收集反馈。
4. 一旦有了初始行为，再尽快切到个性化推荐。

### 9.6.2 如果是新物品冷启动

优先考虑顺序：

1. 用内容特征先把新物品放进候选池。
2. 通过流量扶持给新物品最小曝光机会。
3. 用探索算法控制曝光成本。
4. 收集到初始反馈后，再逐步交给协同过滤或排序模型。

### 9.6.3 如果是整个系统冷启动

优先考虑顺序：

1. 先靠人工运营、规则、精选内容把系统跑起来。
2. 再逐步积累行为数据。
3. 然后再引入更强的个性化推荐。

一个重要原则是：

> 冷启动阶段最重要的目标，不是“一开始就绝对个性化”，而是快速建立反馈闭环。

## 9.7 冷启动策略怎么评估

很多人会问：冷启动效果怎么判断？

可以从 3 个层面看：

### 9.7.1 用户侧指标

- 首次点击率
- 首日留存
- 新用户完成首次消费/播放/阅读的比例

### 9.7.2 物品侧指标

- 新物品曝光率
- 新物品首周获得交互的比例
- 新物品从曝光到首次正反馈的转化率

### 9.7.3 系统侧指标

- 热门集中度是否过高
- 探索带来的损失是否可控
- 冷启动用户转入个性化推荐的速度是否足够快

如果只看点击率，往往会高估热门推荐、低估探索策略的长期价值。

## 9.8 常见误区

### 9.8.1 只靠热门推荐解决一切

热门推荐适合兜底，但如果长期只靠它，系统会越来越偏向头部内容，新物品和长尾内容很难获得机会。

### 9.8.2 一开始就上复杂算法

在冷启动早期，最稀缺的往往不是模型，而是数据。很多时候简单规则、人工精选、兴趣引导比复杂模型更有效。

### 9.8.3 只关注新用户，不关注新物品

很多推荐系统对新用户很敏感，却忽略了新物品冷启动。结果就是系统越来越保守，内容生态越来越弱。

### 9.8.4 把探索理解成“随机乱推”

探索不是无脑随机，而是在可控成本下，给系统争取新信息、给内容争取新机会。

## 9.9 本章小结

### 核心知识点

1. **冷启动类型** — 用户、物品、系统
