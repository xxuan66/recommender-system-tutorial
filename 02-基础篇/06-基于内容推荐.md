# 第六章：基于内容推荐

> 当用户行为还不够多时，如何先利用物品本身的内容特征完成推荐

---

## 6.1 什么是基于内容推荐

### 6.1.1 核心思想

**基于内容的推荐（Content-Based Recommendation）** 的基本思想可以先理解成：

> 根据用户历史喜欢的物品内容特征，推荐具有相似特征的其他物品。

### 6.1.2 与协同过滤的对比

| 维度 | 协同过滤 | 基于内容 |
|------|---------|---------|
| **数据依赖** | 用户评分/行为 | 物品内容特征 |
| **推荐逻辑** | 相似用户/物品 | 相似内容特征 |
| **冷启动** | 无法处理新物品 | 可以处理新物品 |
| **可解释性** | "相似用户也喜欢" | "因为你喜欢 XXX" |
| **多样性** | 较高 | 较低（容易局限） |

### 6.1.3 适用场景

| 场景 | 是否适合 | 原因 |
|------|---------|------|
| **新闻推荐** | ✅ 非常适合 | 文本内容易提取 |
| **音乐推荐** | ✅ 适合 | 音频特征可分析 |
| **视频推荐** | ✅ 适合 | 标签、描述可用 |
| **电商推荐** | ⚠️ 一般 | 需要结合协同过滤 |
| **社交推荐** | ❌ 不适合 | 内容特征不明显 |

### 6.1.4 什么时候优先考虑基于内容推荐

这章最重要的不是背下 TF-IDF 或 Word2Vec，而是知道什么场景下基于内容推荐会比协同过滤更合适。

优先考虑它的典型情况主要有：

1. 新物品很多，而且上线后需要尽快进入推荐系统。
2. 物品本身有比较丰富的内容特征，例如标题、正文、标签、类别、作者、简介。
3. 你需要更强的推荐解释性，能告诉用户“为什么推荐这个内容”。
4. 用户行为数据还不够多，暂时很难稳定跑出协同过滤效果。

但它也不适合所有场景。一个经验判断是：

- 如果内容特征非常强，例如新闻、文章、视频、播客，基于内容通常很自然。
- 如果内容特征弱、但交互行为强，例如纯消费型商品推荐，往往更需要和协同过滤或排序模型结合。
- 如果你希望帮助用户发现“历史兴趣以外的新东西”，单纯基于内容往往不够。

## 6.2 内容特征提取

### 6.2.1 文本特征

**常见文本内容：**
- 新闻：标题、正文
- 商品：名称、描述、类别
- 视频：标题、简介、标签
- 音乐：歌词、流派、专辑信息

#### TF-IDF 特征提取

**TF-IDF（Term Frequency-Inverse Document Frequency）**

**公式：**
```
TF(t, d) = 词 t 在文档 d 中出现的次数 / 文档 d 的总词数
IDF(t) = log(总文档数 / 包含词 t 的文档数)
TF-IDF(t, d) = TF(t, d) × IDF(t)
```

**代码实现：**
```python
from sklearn.feature_extraction.text import TfidfVectorizer
import numpy as np

# 示例文档
documents = [
    "这部电影非常精彩，剧情扣人心弦",
    "这是一部恐怖电影，气氛很紧张",
    "喜剧电影让人捧腹大笑",
    "动作电影场面宏大，特效精彩",
    "剧情电影情感细腻，演技精湛",
]

# TF-IDF 向量化
vectorizer = TfidfVectorizer(
    max_features=100,      # 最大特征数
    stop_words=None,       # 停用词（中文需要自己处理）
    ngram_range=(1, 2),    # 1-2 元语法
)

tfidf_matrix = vectorizer.fit_transform(documents)

print(f"TF-IDF 矩阵形状：{tfidf_matrix.shape}")
print(f"特征词：{vectorizer.get_feature_names_out()[:10]}...")

# 查看某个文档的特征向量
print(f"\n文档 1 的特征向量（前 10 维）：")
print(tfidf_matrix[0].toarray()[0][:10])
```

#### Word2Vec 词向量

**TF-IDF 的局限性：** 无法捕捉词义和上下文

**Word2Vec 解决方案：** 将词映射到低维向量空间

```python

    ["这", "部", "电影", "非常", "精彩"],
    ["剧情", "扣人心弦", "演技", "精湛"],
    ["特效", "精彩", "场面", "宏大"],
    ["喜剧", "电影", "让人", "捧腹大笑"],
]

# 训练 Word2Vec 模型
model = Word2Vec(
    sentences,
    vector_size=50,    # 向量维度
    window=5,          # 上下文窗口
    min_count=1,       # 最小词频
    workers=4          # 线程数
)

# 获取词向量
word_vector = model.wv['电影']
print(f"'电影'的词向量：{word_vector[:10]}...")  # 前 10 维

similarity = model.wv.similarity('精彩', '精湛')

# 文档向量（词向量平均）
def document_vector(doc, model):
    doc_words = [word for word in doc if word in model.wv]
    if len(doc_words) == 0:
        return np.zeros(model.vector_size)
    return np.mean(model.wv[doc_words], axis=0)

doc_vec = document_vector(sentences[0], model)
print(f"文档向量（前 10 维）：{doc_vec[:10]}...")
```

## 6.3 用户画像构建

### 6.3.1 基于历史行为

**基本思想：** 用户喜欢过的物品特征的平均

```python
class UserProfile:
    """用户画像"""
    
    def __init__(self, n_features):
        self.n_features = n_features
        self.preference_vector = np.zeros(n_features)
        self.item_history = []  # (item_id, feature_vector, rating)
    
    def add_interaction(self, item_id, feature_vector, rating=1.0):
        """
        添加用户交互记录
        
        Args:
            item_id: 物品 ID
            feature_vector: 物品特征向量
            rating: 评分/权重（点击=1，购买=2，收藏=3）
        """
        self.item_history.append((item_id, feature_vector, rating))
        self._update_preference()
    
    def _update_preference(self):
        """更新用户偏好向量"""
        if len(self.item_history) == 0:
            return
        
        total_weight = 0
        weighted_sum = np.zeros(self.n_features)
        
        for item_id, feature_vec, rating in self.item_history:
            weighted_sum += feature_vec * rating
            total_weight += rating
        
        if total_weight > 0:
            self.preference_vector = weighted_sum / total_weight
    
    def get_preference(self):
        """获取用户偏好向量"""
        return self.preference_vector
```

### 6.3.2 时间衰减

**考虑用户兴趣随时间变化：**

```python
import time

def time_decay_weight(timestamp, decay_rate=0.1):
    """
    时间衰减权重
    
    Args:
        timestamp: 交互时间戳
        decay_rate: 衰减率（每天）
    
    Returns:
        权重值
    """
    now = time.time()
    days_elapsed = (now - timestamp) / (24 * 3600)
    weight = np.exp(-decay_rate * days_elapsed)
    return weight

# 示例
class UserProfileWithTime:
    """带时间衰减的用户画像"""
    
    def __init__(self, n_features):
        self.n_features = n_features
        self.preference_vector = np.zeros(n_features)
        self.item_history = []
    
    def add_interaction(self, item_id, feature_vector, timestamp=None, rating=1.0):
        if timestamp is None:
            timestamp = time.time()
        
        weight = rating * time_decay_weight(timestamp)
        self.item_history.append((item_id, feature_vector, weight))
        self._update_preference()
    
    def _update_preference(self):
        if len(self.item_history) == 0:
            return
        
        total_weight = 0
        weighted_sum = np.zeros(self.n_features)
        
        for item_id, feature_vec, weight in self.item_history:
            weighted_sum += feature_vec * weight
            total_weight += weight
        
        if total_weight > 0:
            self.preference_vector = weighted_sum / total_weight
```

## 6.4 推荐算法

### 6.4.1 基于相似度

**计算用户画像与物品特征的相似度**

```python
from sklearn.metrics.pairwise import cosine_similarity

class ContentBasedRecommender:
    """基于内容的推荐系统"""
    
    def __init__(self, n_features):
        self.n_features = n_features
        self.user_profiles = {}
        self.item_features = {}
    
    def add_item(self, item_id, feature_vector):
        """添加物品特征"""
        self.item_features[item_id] = feature_vector
    
    def add_user_interaction(self, user_id, item_id, rating=1.0):
        """添加用户交互"""
        if user_id not in self.user_profiles:
            self.user_profiles[user_id] = UserProfile(self.n_features)
        
        feature_vec = self.item_features[item_id]
        self.user_profiles[user_id].add_interaction(item_id, feature_vec, rating)
    
    def recommend(self, user_id, n=10):
        """
        为用户推荐物品
        
        Args:
            user_id: 用户 ID
            n: 推荐数量
        
        Returns:
            推荐列表 [(item_id, score), ...]
        """
        if user_id not in self.user_profiles:
            return []
        
        user_pref = self.user_profiles[user_id].get_preference()
        user_history = set([item_id for item_id, _, _ in self.user_profiles[user_id].item_history])
        
        # 计算所有物品的相似度
        scores = []
        for item_id, feature_vec in self.item_features.items():
            if item_id in user_history:  # 排除已交互物品
                continue
            
            sim = cosine_similarity(
                [user_pref],
                [feature_vec]
            )[0][0]
            
            scores.append((item_id, sim))
        
        # 排序并返回 Top-N
        scores.sort(key=lambda x: x[1], reverse=True)
        return scores[:n]


# 使用示例
if __name__ == "__main__":
    # 创建推荐系统
    n_features = 100  # 特征维度
    recommender = ContentBasedRecommender(n_features)
    
    # 添加物品（示例使用随机特征）
    np.random.seed(42)
    for i in range(20):
        feature_vec = np.random.rand(n_features)
        recommender.add_item(i, feature_vec)
    
    # 添加用户交互
    user_id = 0
    recommender.add_user_interaction(user_id, 0, rating=5)
    recommender.add_user_interaction(user_id, 1, rating=4)
    recommender.add_user_interaction(user_id, 2, rating=3)
    
    # 推荐
    recommendations = recommender.recommend(user_id, n=5)
    
    print(f"为用户 {user_id} 推荐的物品：")
    for item_id, score in recommendations:
        print(f"  物品 {item_id}: 相似度 {score:.4f}")
```

### 6.4.2 基于分类模型

**将推荐视为二分类问题（喜欢/不喜欢）**

```python
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier

class ClassificationRecommender:
    """基于分类模型的推荐"""
    
    def __init__(self):
        self.model = LogisticRegression()
        self.user_models = {}
    
    def train(self, user_id, positive_features, negative_features):
        """
        训练用户模型
        
        Args:
            user_id: 用户 ID
            positive_features: 正样本特征（喜欢的物品）
            negative_features: 负样本特征（不喜欢的物品）
        """
        X = np.vstack([positive_features, negative_features])
        y = np.array([1] * len(positive_features) + [0] * len(negative_features))
        
        model = LogisticRegression()
        model.fit(X, y)
        
        self.user_models[user_id] = model
    
    def predict(self, user_id, item_features):
        """预测用户是否喜欢物品"""
        if user_id not in self.user_models:
            return 0.5
        
        model = self.user_models[user_id]
        proba = model.predict_proba([item_features])[0][1]
        return proba
    
    def recommend(self, user_id, candidate_features, n=10):
        """推荐 Top-N 物品"""
        scores = []
        for item_id, features in candidate_features.items():
            score = self.predict(user_id, features)
            scores.append((item_id, score))
        
        scores.sort(key=lambda x: x[1], reverse=True)
        return scores[:n]
```

## 6.5 完整实战案例

### 6.5.1 新闻推荐系统

```python
import pandas as pd
import numpy as np
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

class NewsRecommender:
    """新闻推荐系统"""
    
    def __init__(self):
        self.tfidf = TfidfVectorizer(
            max_features=1000,
            stop_words='english',  # 中文需要自定义停用词
            ngram_range=(1, 2),
        )
        self.news_features = {}
        self.user_profiles = {}
        self.is_fitted = False
    
    def add_news(self, news_id, title, content, category):
        """添加新闻"""
        # 组合文本
        text = f"{title} {content} {category}"
        
        if news_id not in self.news_features:
            self.news_features[news_id] = {
                'text': text,
                'category': category,
                'title': title,
            }
    
    def fit(self):
        """训练 TF-IDF 模型"""
        texts = [info['text'] for info in self.news_features.values()]
        self.tfidf_matrix = self.tfidf.fit_transform(texts)
        self.is_fitted = True
        
        # 建立 ID 映射
        self.news_ids = list(self.news_features.keys())
        self.id_to_idx = {news_id: idx for idx, news_id in enumerate(self.news_ids)}
    
    def add_user_click(self, user_id, news_id):
        """记录用户点击"""
        if user_id not in self.user_profiles:
            self.user_profiles[user_id] = []
        
        if news_id in self.id_to_idx:
            self.user_profiles[user_id].append(news_id)
    
    def recommend(self, user_id, n=10):
        """为用户推荐新闻"""
        if not self.is_fitted:
            self.fit()
        
        if user_id not in self.user_profiles:
            # 冷启动：推荐热门新闻
            return self._popular_news(n)
        
        # 获取用户历史点击的新闻
        clicked_news = self.user_profiles[user_id]
        if len(clicked_news) == 0:
            return self._popular_news(n)
        
        # 计算用户偏好向量（历史点击新闻的平均 TF-IDF）
        clicked_indices = [self.id_to_idx[nid] for nid in clicked_news]
        user_vector = np.mean(self.tfidf_matrix[clicked_indices].toarray(), axis=0)
        
        # 计算与所有新闻的相似度
        similarities = cosine_similarity([user_vector], self.tfidf_matrix)[0]
        
        # 排除已点击的新闻
        for nid in clicked_news:
            idx = self.id_to_idx[nid]
            similarities[idx] = -1
        
        # 获取 Top-N
        top_indices = np.argsort(similarities)[::-1][:n]
        recommendations = [(self.news_ids[idx], similarities[idx]) for idx in top_indices]
        
        return recommendations
    
    def _popular_news(self, n=10):
        """返回热门新闻（简单实现：按 ID 排序）"""
        return [(news_id, 0) for news_id in list(self.news_features.keys())[:n]]


# 使用示例
if __name__ == "__main__":
    # 创建推荐系统
    recommender = NewsRecommender()
    
    # 添加新闻
    recommender.add_news(1, "AI 技术突破", "人工智能领域取得新进展...", "科技")
    recommender.add_news(2, "足球比赛结果", "昨晚的足球比赛...", "体育")
    recommender.add_news(3, "股市分析", "今日股市行情分析...", "财经")
    recommender.add_news(4, "AI 应用案例", "人工智能在医疗领域的应用...", "科技")
    recommender.add_news(5, "篮球赛事", "NBA 比赛精彩瞬间...", "体育")
    
    # 训练
    recommender.fit()
    
    # 模拟用户行为
    user_id = "user_001"
    recommender.add_user_click(user_id, 1)  # 点击 AI 新闻
    recommender.add_user_click(user_id, 4)  # 点击 AI 应用新闻
    
    # 推荐
    recommendations = recommender.recommend(user_id, n=3)
    
    print(f"为用户 {user_id} 推荐的新闻：")
    for news_id, score in recommendations:
        news_info = recommender.news_features[news_id]
        print(f"  [{news_info['category']}] {news_info['title']} (相似度：{score:.4f})")
```

## 6.6 优缺点分析

### 6.6.1 优点

| 优点 | 说明 |
|------|------|
| **无冷启动** | 新物品可以立即推荐 |
| **可解释性强** | "因为你喜欢 XXX，所以推荐 YYY" |
| **用户独立** | 不依赖其他用户数据 |
| **透明度高** | 推荐逻辑清晰可见 |

### 6.6.2 缺点

| 缺点 | 说明 | 解决方案 |
|------|------|---------|
| **内容分析要求高** | 需要提取有效特征 | 使用深度学习 |
| **推荐多样性低** | 容易局限在同类内容 | 引入探索机制 |
| **无法发现新兴趣** | 只能基于历史推荐 | 结合协同过滤 |
| **特征工程复杂** | 需要领域知识 | 使用预训练模型 |

### 6.6.3 混合推荐

**结合协同过滤和基于内容：**

```python
class HybridRecommender:
    """混合推荐系统"""
    
    def __init__(self, cf_weight=0.5, cb_weight=0.5):
        self.cf_recommender = None  # 协同过滤
        self.cb_recommender = None  # 基于内容
        self.cf_weight = cf_weight
        self.cb_weight = cb_weight
    
    def recommend(self, user_id, n=10):
        """混合推荐"""
        cf_recs = self.cf_recommender.recommend(user_id, n)
        cb_recs = self.cb_recommender.recommend(user_id, n)
        
        # 融合推荐结果
        combined = {}
        for item_id, score in cf_recs:
            combined[item_id] = score * self.cf_weight
        for item_id, score in cb_recs:
            if item_id in combined:
                combined[item_id] += score * self.cb_weight
            else:
                combined[item_id] = score * self.cb_weight
        
        # 排序
        sorted_recs = sorted(combined.items(), key=lambda x: x[1], reverse=True)
        return sorted_recs[:n]
```

## 6.7 一个最小实验该怎么做

如果你准备把这章知识落成一个最小实验，推荐按下面顺序推进：

### 6.7.1 第一步：先确定特征来源

先回答一个关键问题：

> 你打算用什么内容特征来代表物品？

常见选择：

- 新闻：标题、正文、栏目、关键词。
- 视频：标题、简介、标签、作者、时长。
- 商品：标题、类目、品牌、描述、价格区间。

### 6.7.2 第二步：先做一个简单 baseline

不要一开始就上复杂深度模型，可以先做：

1. 用 TF-IDF 提取文本特征。
2. 用用户历史点击内容做平均，形成用户画像。
3. 用余弦相似度做 Top-N 推荐。

### 6.7.3 第三步：固定评估方式再做改进

建议至少比较下面几类变化：

- TF-IDF vs 更复杂的语义向量。
- 是否加入类别、标签等结构化特征。
- 是否加入时间衰减。
- 是否加入探索策略提升多样性。

如果你每次同时改很多东西，最后通常很难知道到底是哪一步带来了效果变化。

### 6.7.4 一个实用检查清单

在说“我实现了基于内容推荐”之前，最好确认：

- 你的物品特征到底是什么。
- 你的用户画像如何构建。
- 你的相似度函数为什么这样选。
- 你如何处理用户兴趣变化。
- 你如何评估多样性和新颖性问题。

## 6.8 常见误区

### 6.8.1 只要有文本，就一定适合基于内容推荐

并不是。内容特征很多不代表内容特征一定有效。如果文本噪声很大、标签质量差，推荐效果可能依然不稳定。

### 6.8.2 认为相似度高就等于推荐质量高

相似度高通常意味着“更像历史偏好”，但不一定意味着“用户真的更想点”。相似度只是信号，不是最终目标。

### 6.8.3 忽略多样性问题

基于内容推荐很容易把用户锁在熟悉内容里，越推越窄。如果没有探索机制，用户的视野会越来越单一。

### 6.8.4 把基于内容推荐当成协同过滤的替代品

很多真实系统里，它更像是协同过滤和排序模型的补充，尤其在新物品冷启动阶段价值更大。
